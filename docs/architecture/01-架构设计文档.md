# 都达云台 (DudaNexus) 微服务架构设计文档

## 1. 架构总览

### 1.1 项目定位
DudaNexus是一个基于Spring Cloud技术栈构建的企业级微服务平台，旨在为都达网络科技有限公司提供统一的技术底座，支撑多端应用（Web、App、小程序）的业务需求。

### 1.2 核心业务域
- **电商平台**：商品管理、订单交易、支付结算
- **视频直播**：直播推流、实时互动、礼物打赏
- **社交平台**：动态发布、评论点赞、关注关系
- **即时通讯**：单聊群聊、消息推送、文件传输

## 3. 微服务拆分

### 3.1 服务拆分原则
1. **单一职责**：每个服务只负责一个业务域
2. **高内聚低耦合**：服务内部高度内聚，服务之间松散耦合
3. **业务边界清晰**：按业务领域划分（DDD）
4. **独立部署**：每个服务可独立开发、测试、部署
5. **数据独立**：每个服务拥有独立数据库

## 4. 数据库设计原则

### 4.1 数据库分库策略
按业务域分库：
- `dn_auth`：认证授权库
- `dn_user`：用户中心库
- `dn_product`：商品库
- `dn_order`：订单库
- `dn_payment`：支付库
- `dn_im`：即时通讯库
- `dn_live`：直播库
- `dn_social`：社交库
- `dn_content`：内容库

### 4.2 分表策略
- **水平分表**：数据量大时按ID或时间分表
- **垂直分表**：将大表拆分为常用字段和不常用字段

**示例：**
```sql
-- 订单表按月分表
dn_order_202401
dn_order_202402
dn_order_202403

-- 日志表按日分表
dn_log_20240307
dn_log_20240308
```

### 4.3 数据一致性
- **强一致性场景**：使用Seata分布式事务
- **最终一致性场景**：使用MQ+本地消息表
- **读多写少场景**：使用读写分离+缓存

## 5. 接口设计规范

### 5.1 RESTful API设计
```
GET    /api/v1/{resource}/{id}      # 获取单个资源
GET    /api/v1/{resource}           # 获取资源列表
POST   /api/v1/{resource}           # 创建资源
PUT    /api/v1/{resource}/{id}      # 更新资源
DELETE /api/v1/{resource}/{id}      # 删除资源
PATCH  /api/v1/{resource}/{id}      # 部分更新
```

### 5.2 统一响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1709769600000,
  "traceId": "trace-id-123"
}
```

### 5.3 错误码规范
```
# 通用错误码
200  成功
400  请求参数错误
401  未认证
403  无权限
404  资源不存在
500  服务器内部错误

# 业务错误码
10001 用户不存在
10002 用户已存在
10003 密码错误
20001 订单不存在
20002 订单状态异常
30001 商品库存不足
```

## 6. 安全设计

### 6.1 认证授权
- **认证方式**：JWT + Refresh Token
- **授权模型**：RBAC（角色-权限）
- **SSO集成**：支持OAuth2.0

### 6.2 数据安全
- **传输加密**：HTTPS/TLS
- **敏感数据加密**：AES-256
- **密码存储**：BCrypt哈希
- **SQL注入防护**：MyBatis预编译

### 6.3 接口安全
- **接口签名**：请求签名验证
- **防重放攻击**：时间戳+nonce
- **限流防刷**：Redis + 令牌桶算法
- **XSS防护**：输入输出过滤

## 7. 高可用设计

### 7.1 服务高可用
- **服务集群**：无状态服务多实例部署
- **负载均衡**：Nginx + Ribbon
- **健康检查**：Nacos健康检查
- **故障隔离**：熔断降级

### 7.2 数据库高可用
- **MySQL主从复制**：一主多从
- **读写分离**：ShardingSphere
- **Redis哨兵**：高可用集群
- **数据备份**：定时备份+binlog

### 7.3 限流降级
```java
// 接口限流
@SentinelResource(value = "getUser", blockHandler = "handleBlock")
public Result<UserVO> getUser(Long id) {
    // ...
}

// 熔断降级
@FeignClient(name = "duda-user-service", fallback = UserServiceFallback.class)
public interface UserService {
    // ...
}
```

## 8. 性能优化

### 8.1 缓存策略
- **多级缓存**：本地缓存 + Redis
- **缓存预热**：应用启动时加载热点数据
- **缓存更新**：Cache Aside模式
- **缓存穿透**：布隆过滤器
- **缓存雪崩**：随机过期时间

### 8.2 异步处理
- **异步解耦**：RocketMQ
- **线程池隔离**：业务线程池
- **CompletableFuture**：异步编程

### 8.3 数据库优化
- **索引优化**：合理创建索引
- **SQL优化**：避免全表扫描
- **分页优化**：延迟关联
- **批量操作**：批量插入/更新
