# 模块命名重构计划

## 📋 重构目标

将当前模块重命名为标准的接口-实现分离模式：
- 接口模块：`{业务名}-interface`
- 实现模块：`{业务名}-provider`

## 🔧 重构步骤

### 第 1 步：重命名目录

```bash
# 认证模块
mv duda-auth/duda-auth-api duda-auth/duda-auth-interface
mv duda-auth/duda-auth-service duda-auth/duda-auth-provider

# 用户中心
mv duda-usercenter/duda-user-service duda-usercenter/duda-user-interface
mkdir duda-usercenter/duda-user-provider
# ... 重复其他模块
```

### 第 2 步：更新 pom.xml

#### 父 pom.xml 模块列表

```xml
<modules>
    <!-- 认证授权 -->
    <module>duda-auth/duda-auth-interface</module>
    <module>duda-auth/duda-auth-provider</module>

    <!-- 用户中心 -->
    <module>duda-usercenter/duda-user-interface</module>
    <module>duda-usercenter/duda-user-provider</module>
    <module>duda-usercenter/duda-profile-interface</module>
    <module>duda-usercenter/duda-profile-provider</module>
    <!-- ... -->
</modules>
```

#### provider 模块依赖 interface

```xml
<!-- duda-user-provider/pom.xml -->
<dependencies>
    <!-- 依赖接口模块 -->
    <dependency>
        <groupId>com.duda</groupId>
        <artifactId>duda-user-interface</artifactId>
        <version>${project.version}</version>
    </dependency>

    <!-- 其他依赖 -->
</dependencies>
```

### 第 3 步：拆分代码

#### interface 模块包含：

```
duda-user-interface/
├── pom.xml                 # 轻量依赖（Lombok + OpenFeign）
└── src/main/java/
    └── com/duda/user/api/
        ├── dto/            # 数据传输对象
        │   ├── UserDTO.java
        │   └── UserLoginDTO.java
        ├── constants/      # 常量定义
        │   └── UserConstants.java
        └── feign/          # OpenFeign 接口
            └── UserClient.java
```

#### provider 模块包含：

```
duda-user-provider/
├── pom.xml                 # 重量依赖（所有框架）
└── src/main/java/
    └── com/duda/user/
        ├── controller/     # REST 控制器
        ├── service/        # 业务接口
        │   └── impl/       # 业务实现
        ├── mapper/         # MyBatis Mapper
        │   └── UserMapper.java
        ├── entity/         # 数据库实体
        │   └── UserPO.java
        └── config/         # 配置类
```

### 第 4 步：调整导入

```java
// provider 模块中调整导入
// 从：import com.duda.user.dto.UserDTO;
// 改为：import com.duda.user.api.dto.UserDTO;
```

## ✅ 命名规范

### 接口模块 (interface)

**命名规则：** `duda-{业务名}-interface`

**示例：**
- `duda-user-interface` - 用户服务接口
- `duda-order-interface` - 订单服务接口
- `duda-payment-interface` - 支付服务接口

**特点：**
- 轻量级依赖
- 只包含 DTO、常量、Feign 接口
- 不包含业务实现

### 实现模块 (provider)

**命名规则：** `duda-{业务名}-provider`

**示例：**
- `duda-user-provider` - 用户服务实现
- `duda-order-provider` - 订单服务实现
- `duda-payment-provider` - 支付服务实现

**特点：**
- 重量级依赖
- 包含所有业务逻辑
- 依赖对应的 interface 模块

## 📝 优势

### 1. 一目了然

看到模块名称就知道是一对：

```
duda-user-interface     ← 接口
duda-user-provider      ← 实现
```

### 2. 快速定位

```
duda-user-interface/    # 接口定义在这里
duda-user-provider/     # 实现代码在这里
```

### 3. 与参考项目一致

```
qiyu-live-user-interface    # 参考项目
qiyu-live-user-provider     # 参考项目

duda-user-interface         # DudaNexus（命名一致）
duda-user-provider          # DudaNexus（命名一致）
```

## 🎯 执行建议

### 选项 1：全量重构（推荐用于项目初期）

一次性重构所有模块，适合项目刚开始开发。

**时间：** 1-2 小时

### 选项 2：渐进式重构（推荐用于项目中期）

先重构核心模块，其他模块逐步重构。

**优先级：**
1. duda-auth（认证）
2. duda-user（用户）
3. duda-order（订单）
4. duda-payment（支付）
5. 其他模块

### 选项 3：新建模块直接使用规范

已有模块暂时不重构，新模块直接使用新规范。
