# 公共模块错误修复说明

## 🐛 发现的错误

### 1. BeanUtils 命名冲突

**错误信息：**
```
在 BeanUtils 类中调用 BeanUtils.copyProperties() 导致递归调用
```

**原因：**
- 我定义的工具类名 `BeanUtils` 与 Spring 的 `BeanUtils` 类名冲突
- 导致方法内部调用 `BeanUtils.copyProperties()` 时递归调用自己的方法

**解决方案：**
- 将工具类重命名为 `BeanCopyUtils`
- 使用 Spring 的 `BeanUtils.copyProperties()` 时使用全限定名 `org.springframework.beans.BeanUtils`

**修改文件：**
- 删除：`duda-common-core/src/main/java/com/duda/common/util/BeanUtils.java`
- 新增：`duda-common-core/src/main/java/com/duda/common/util/BeanCopyUtils.java`

---

### 2. 缺少 Swagger 注解依赖

**错误信息：**
```
无法解析符号 'Schema'
```

**原因：**
- `PageQuery` 类使用了 `@Schema` 注解（Swagger 注解）
- 但 `duda-common-core` 的 pom.xml 中没有添加 Swagger 依赖

**解决方案：**
在 `duda-common-core/pom.xml` 中添加：
```xml
<dependency>
    <groupId>io.swagger.core.v3</groupId>
    <artifactId>swagger-annotations</artifactId>
    <version>2.2.8</version>
</dependency>
```

---

### 3. 缺少 MyBatis-Plus 注解依赖

**错误信息：**
```
无法解析符号 'TableId'
无法解析符号 'TableName'
无法解析符号 'TableField'
```

**原因：**
- `BaseEntity` 类使用了 MyBatis-Plus 的注解
- 但 `duda-common-core` 的 pom.xml 中没有添加 MyBatis-Plus 注解依赖

**解决方案：**
在 `duda-common-core/pom.xml` 中添加：
```xml
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-annotation</artifactId>
    <version>3.5.5</version>
</dependency>
```

---

### 4. 缺少 Spring Beans 依赖

**错误信息：**
```
无法解析符号 'BeanUtils'（Spring 的）
```

**原因：**
- `BeanCopyUtils` 使用了 Spring 的 `BeanUtils.copyProperties()`
- 但没有显式声明 Spring Beans 依赖

**解决方案：**
在 `duda-common-core/pom.xml` 中添加：
```xml
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-beans</artifactId>
</dependency>
```

---

## ✅ 已修复的问题

### 1. duda-common-core/pom.xml

添加了以下依赖：
```xml
<!-- MyBatis-Plus 注解 -->
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-annotation</artifactId>
    <version>3.5.5</version>
</dependency>

<!-- Spring Beans -->
<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-beans</artifactId>
</dependency>

<!-- Swagger注解 -->
<dependency>
    <groupId>io.swagger.core.v3</groupId>
    <artifactId>swagger-annotations</artifactId>
    <version>2.2.8</version>
</dependency>

<!-- Jackson -->
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-annotations</artifactId>
</dependency>
```

### 2. BeanCopyUtils

新建了 `BeanCopyUtils` 类，替代原来的 `BeanUtils`：
- 使用 `org.springframework.beans.BeanUtils` 的全限定名调用 Spring 的方法
- 避免命名冲突

## 📝 影响范围

### 需要修改的代码

如果你已经按照之前的指南写了代码，需要做以下修改：

**原来：**
```java
import com.duda.common.util.BeanUtils;

UserDTO dto = BeanUtils.copy(userPO, UserDTO.class);
```

**修改为：**
```java
import com.duda.common.util.BeanCopyUtils;

UserDTO dto = BeanCopyUtils.copy(userPO, UserDTO.class);
```

**只需修改包名和类名，其他使用方式完全相同！**
