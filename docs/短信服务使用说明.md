# 短信服务使用说明

## 📌 服务概述

`duda-msg` 是一个支持开发/生产环境插拔式切换的短信服务模块。

### 核心特性

1. **插拔式设计** - 开发环境不花钱，生产环境真实发送
2. **防重发机制** - 60秒内不能重复发送
3. **Redis缓存** - 验证码60秒过期
4. **验证后删除** - 防止验证码重复使用

#### 步骤1：前端 - 用户输入手机号

```

│  ┌───────────────────────────┐  │

│  │        [注册]             │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

#### 步骤2：前端 - 点击"获取验证码"

```javascript
// 前端调用发送验证码接口
async function getSmsCode() {
    const phone = document.getElementById('phone').value;

    const response = await fetch('/api/auth/sms/send?phone=' + phone, {
        method: 'POST'
    });

    const result = await response.json();

    if (result.code === 200) {
        alert('验证码已发送');
        // 开始60秒倒计时
        startCountdown(60);
    }
}
```

#### 步骤3：后端 - 发送验证码

```bash
# 请求
POST /api/auth/sms/send?phone=13800138000

# 响应（开发环境）
{
  "code": 200,
  "message": "success",
  "data": "验证码发送成功，请查收短信"
}
```

**开发环境日志输出**：
```
2026-03-12 01:00:00.000 [main] INFO  SmsServiceImpl - 【sendLoginCode】开发环境，验证码打印到日志，phone=13800138000, code=5234
========================================
【验证码】手机号：13800138000，验证码：5234
========================================
```

#### 步骤4：前端 - 用户输入验证码并注册

```javascript
// 用户输入验证码后，调用注册接口
async function register() {
    const phone = '13800138000';
    const code = '5234';  // 用户从日志中看到的验证码
    const realName = '张三';

    const response = await fetch('/api/auth/register', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json'
        },
        body: JSON.stringify({
            userType: 'normal',
            registerType: 'phone_sms',
            phone: phone,
            phoneVerifyCode: code,
            realName: realName
        })
    });

    const result = await response.json();
    console.log(result);
}
```

#### 步骤5：后端 - 验证码校验

```java
// AuthServiceImpl 会调用短信服务RPC
MsgCheckDTO checkResult = smsRpc.checkLoginCode(phone, code);

if (checkResult.isCheckStatus()) {
    // 验证码正确，继续注册流程
    // ...
}
```

## ⚙️ 配置说明

### 开发环境配置

```yaml
duda:
  sms:
    real-send: false  # ⭐ 关键：false表示开发环境
```

**行为**：
- ✅ 生成验证码并存入Redis
- ✅ 验证码打印到日志
- ❌ **不调用真实短信服务**
- 💰 **不花钱**

### 生产环境配置

```yaml
duda:
  sms:
    real-send: true  # ⭐ 关键：true表示生产环境
```

**行为**：
- ✅ 生成验证码并存入Redis
- ✅ 调用真实短信服务（容联云/阿里云/腾讯云）
- ✅ 发送到用户手机
- 💰 **按实际发送量计费**

## 🔌 API接口文档

### 1. 发送验证码

**接口**：`POST /api/auth/sms/send`

**参数**：
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| phone | String | 是 | 手机号（11位） |

**响应**：
```json
{
  "code": 200,
  "message": "success",
  "data": "验证码发送成功，请查收短信"
}
```

**错误情况**：
- 手机号格式错误
- 60秒内重复发送

### 2. 使用验证码注册

**接口**：`POST /api/auth/register`

**请求体**：
```json
{
  "userType": "normal",
  "registerType": "phone_sms",
  "phone": "13800138000",
  "phoneVerifyCode": "1234",
  "realName": "张三"
}
```

**响应**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": 1000123456789,
    "username": "138****8000",
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "tokenType": "Bearer",
    "expiresIn": 900
  }
}
```

## 📝 常见问题

### Q1: 开发环境为什么收不到短信？

**A**: 开发环境设置了 `real-send: false`，验证码只打印到日志，不会真实发送。这是为了节省开发测试费用。

**查看验证码**：
```bash
# 查看日志
tail -f /path/to/duda-msg-provider.log | grep "验证码"
```

### Q2: 如何切换到生产环境？

**A**: 修改Nacos配置：
```yaml
duda:
  sms:
    real-send: true  # 改为true
```

### Q3: 验证码有效期多久？

**A**: 默认60秒。可在配置中修改：
```yaml
duda:
  sms:
    code-expire-seconds: 60  # 修改这个值
```

### Q4: 验证码长度是几位？

**A**: 默认4位。可在配置中修改：
```yaml
duda:
  sms:
    code-length: 4  # 修改这个值
```

### Q5: 60秒内不能重复发送？

**A**: 是的，这是防重发机制。防止恶意刷短信接口。
