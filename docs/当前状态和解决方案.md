### 方案 2: 修复 Redis 连接

**问题：** Redis 连接失败

**可能原因和解决方案：**

#### 原因 1: Redis 密码不正确

**测试：**
```bash
# 测试无密码连接
redis-cli -h 120.26.170.213 -p 6379 ping

# 测试有密码连接
redis-cli -h 120.26.170.213 -p 6379 -a duda2024 ping
```

**如果上面命令成功：**
- 说明密码可能不对或没有设置密码
- 在 Nacos 配置中更新 Redis 密码配置

#### 原因 2: Redis 服务未启动

**检查：**
```bash
# 查看 Redis 容器状态
ssh root@120.26.170.213

# 如果没有运行，启动它
docker start duda-redis
```

**检查：**

**临时解决方案：**
- 如果服务启动时提示 Redis 连接失败，可以先在配置中注释掉 Redis
- 等服务启动后再修复 Redis 问题

### 方案 3: 释放端口 8080

**问题：** 端口 8080 被 QQ 和 Python 占用

**解决方法：**

#### 方法 A: 关闭占用端口的进程（推荐）

```bash
# 查看占用端口的进程
lsof -i:8080

# 关闭指定进程
kill -9 <PID>

# 或者一次性关闭所有占用 8080 的进程
kill -9 $(lsof -ti:8080)
```

#### 方法 B: 修改网关端口

如果不想关闭 QQ，可以修改网关端口：

在 Nacos 的 `duda-gateway-dev.yml` 配置中：

```yaml
server:
  port: 8888  # 改成其他端口，如 8888
```

或者在 `bootstrap.yml` 中覆盖：
```yaml
server:
  port: ${SERVER_PORT:8888}
```
