# DudaNexus 技术栈详解

## 📋 核心框架

### Spring Boot 3.2.0

**版本说明：**
- Spring Boot 3.x 是基于 Jakarta EE 9+ 的版本
- 需要至少 Java 17
- 使用 `javax.*` → `jakarta.*` 命名空间迁移

**主要特性：**
- ✅ 原生镜像支持（GraalVM）
- ✅ 虚拟线程支持（Java 21+）
- ✅ 观测性改进
- ✅ 性能优化

**关键依赖：**
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
```

### Java 17 (LTS)

**版本说明：**
- Java 17 是长期支持版本（LTS）
- 支持到 2029 年
- 性能相比 Java 8 提升显著

**主要特性：**
- ✅ Record（记录类）
- ✅ Pattern Matching（模式匹配）
- ✅ Sealed Classes（密封类）
- ✅ Text Blocks（文本块）
- ✅ Switch Expressions（Switch 表达式）

**为什么选择 Java 17：**
1. Spring Boot 3.x 强制要求
2. 性能提升 10-15%
3. 新特性提高开发效率
4. 长期支持，稳定可靠

## 🔐 安全与认证

### JJWT 0.12.3

**说明：**
- JWT (JSON Web Token) 库
- 创建和验证 JWT

**主要特性：**
- ✅ 创建 JWT
- ✅ 验证 JWT
- ✅ 解析 JWT

**使用示例：**
```java
// 创建 JWT
String jwt = Jwts.builder()
    .subject(userId)
    .claim("username", username)
    .issuedAt(new Date())
    .expiration(Date.from(Instant.now().plus(7, ChronoUnit.DAYS)))
    .signWith(secretKey)
    .compact();

// 验证 JWT
Claims claims = Jwts.parser()
    .verifyWith(secretKey)
    .build()
    .parseSignedClaims(jwt)
    .getPayload();
```

## ⚠️ 注意事项

### 1. Spring Cloud Alibaba RC 版本

```xml
<!-- 当前使用的是 RC 版本 -->
<spring-cloud-alibaba.version>2023.0.0.0-RC1</spring-cloud-alibaba.version>
```

**建议：**
- ⚠️ RC 版本相对稳定，但可能有 bug
- ✅ 等待正式版发布后升级
- ✅ 关注官方更新日志

### 2. Jakarta EE 迁移

```java
// ❌ 旧版 (Spring Boot 2.x)
import javax.servlet.*;
import javax.persistence.*;

// ✅ 新版 (Spring Boot 3.x)
import jakarta.servlet.*;
import jakarta.persistence.*;
```

### 3. Java 17 强制要求

- Spring Boot 3.x 必须使用 Java 17+
- 无法降级到 Java 8
- 老项目需要升级 JDK
