## 2. Docker部署

### 2.1 Dockerfile规范

```dockerfile
# 多阶段构建
# 构建阶段
FROM maven:3.9-openjdk-17-slim AS builder
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

# 运行阶段
FROM openjdk:17-jdk-slim

# 设置时区
ENV TZ=Asia/Shanghai
RUN ln -snf /usr/share/zoneinfo/$TZ /etc/localtime && echo $TZ > /etc/timezone

# 创建应用用户
RUN groupadd -r duda && useradd -r -g duda duda

# 创建日志目录
RUN mkdir -p /data/logs && chown -R duda:duda /data

# 设置工作目录
WORKDIR /app

# 复制JAR文件
COPY --from=builder /app/target/*.jar app.jar

# 修改文件所有者
RUN chown -R duda:duda /app

# 切换用户
USER duda

# 暴露端口
EXPOSE 8080

# JVM参数
ENV JAVA_OPTS="-Xms512m -Xmx1024m -XX:+UseG1GC -XX:MaxGCPauseMillis=200"

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=60s --retries=3 \
  CMD curl -f http://localhost:8080/actuator/health || exit 1

# 启动应用
ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar app.jar"]
```

## 4. CI/CD流程

### 4.1 Jenkins Pipeline

```groovy
pipeline {
    agent any

    environment {
        DOCKER_REGISTRY = 'registry.duda.com'
        DOCKER_CREDENTIALS = credentials('docker-registry-creds')
        GIT_CREDENTIALS = credentials('git-creds')
        KUBECONFIG = credentials('kubeconfig')
    }

    stages {
        stage('Checkout') {
            steps {
                git url: 'https://github.com/duda/DudaNexus.git',
                    branch: 'main',
                    credentialsId: "${GIT_CREDENTIALS}"
            }
        }

        stage('Build') {
            steps {
                sh 'mvn clean package -DskipTests'
            }
        }

        stage('Test') {
            steps {
                sh 'mvn test'
            }
            post {
                always {
                    junit 'target/surefire-reports/*.xml'
                }
            }
        }

        stage('SonarQube') {
            steps {
                sh 'mvn sonar:sonar'
            }
        }

        stage('Build Image') {
            steps {
                script {
                    def appImage = docker.build("${DOCKER_REGISTRY}/duda-user-service:${BUILD_NUMBER}")
                    appImage.push()
                    appImage.push('latest')
                }
            }
        }

        stage('Deploy to Test') {
            steps {
                sh """
                    kubectl set image deployment/duda-user-service \
                        duda-user-service=${DOCKER_REGISTRY}/duda-user-service:${BUILD_NUMBER} \
                        -n duda-test
                """
            }
        }

        stage('Verify Deployment') {
            steps {
                sh 'kubectl rollout status deployment/duda-user-service -n duda-test'
            }
        }

        stage('Deploy to Prod') {
            when {
                branch 'main'
            }
            input {
                message 'Deploy to production?'
            }
            steps {
                sh """
                    kubectl set image deployment/duda-user-service \
                        duda-user-service=${DOCKER_REGISTRY}/duda-user-service:${BUILD_NUMBER} \
                        -n duda-prod
                """
            }
        }
    }

    post {
        success {
            echo 'Deployment successful!'
        }
        failure {
            echo 'Deployment failed!'
        }
    }
}
```

### 5.3 告警规则

```yaml
groups:
- name: duda-alerts
  rules:
  # 服务不可用告警
  - alert: ServiceDown
    expr: up{job=~"duda-.*"} == 0
    for: 1m
    labels:
      severity: critical
    annotations:
      summary: "Service {{ $labels.instance }} is down"

  # 高错误率告警
  - alert: HighErrorRate
    expr: rate(http_requests_total{status=~"5.."}[5m]) > 0.01
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "High error rate on {{ $labels.instance }}"

  # 高内存使用告警
  - alert: HighMemoryUsage
    expr: jvm_memory_used_bytes{area="heap"} / jvm_memory_max_bytes{area="heap"} > 0.9
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "High memory usage on {{ $labels.instance }}"

  # CPU使用率告警
  - alert: HighCPUUsage
    expr: rate(process_cpu_seconds_total[5m]) * 100 > 80
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "High CPU usage on {{ $labels.instance }}"
```

## 7. 备份策略

### 7.1 数据备份

**MySQL备份：**
```bash
# 全量备份脚本
#!/bin/bash
BACKUP_DIR=/data/backup/mysql
DATE=$(date +%Y%m%d)

# 全量备份
mysqldump -u root -p${MYSQL_PASSWORD} \
  --single-transaction \
  --routines \
  --triggers \
  --events \
  --all-databases > ${BACKUP_DIR}/full_backup_${DATE}.sql

# 压缩备份文件
gzip ${BACKUP_DIR}/full_backup_${DATE}.sql

# 删除7天前的备份
find ${BACKUP_DIR} -name "*.sql.gz" -mtime +7 -delete
```

**Redis备份：**
```bash
#!/bin/bash
# RDB备份
cp /var/lib/redis/dump.rdb /data/backup/redis/dump_$(date +%Y%m%d_%H%M%S).rdb
```

## 8. 应急预案

### 8.1 服务故障

**发现：**
- 监控告警
- 用户反馈

**处理流程：**
1. 确认故障范围
2. 评估影响程度
3. 启动应急预案
4. 定位问题
5. 实施修复
6. 验证恢复
7. 复盘总结

### 8.2 数据库故障

**主库故障：**
1. 提升从库为主库
2. 修改应用连接配置
3. 重建主从复制

**数据丢失：**
1. 停止应用
2. 评估数据丢失范围
3. 从备份恢复
4. 重放binlog
5. 验证数据
6. 恢复服务

### 8.3 缓存故障

**Redis故障：**
1. 检查Redis状态
2. 尝试重启
3. 切换到从库
4. 降级处理（走数据库）
