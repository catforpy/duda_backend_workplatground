## 📝 测试步骤

### 第一步：在 Nacos 创建最小配置

为了快速测试，我们先创建最小配置集：

#### 1. 创建命名空间

登录 Nacos: http://120.26.170.213:8848/nacos

创建命名空间：
```
ID: duda-dev
名称: 开发环境
```

#### 2. 创建公共配置

切换到 `duda-dev` 命名空间，创建配置：

```
Data ID: common-dev.yml
Group: DEFAULT_GROUP
配置格式: YAML
```

**配置内容（最小版本）：**

```yaml
# 最小公共配置
spring:
  data:
    redis:
      host: 120.26.170.213
      port: 6379
      password: duda2024
      database: 0

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true

management:
  endpoints:
    web:
      exposure:
        include: '*'
```

#### 3. 创建认证服务配置

```
Data ID: duda-auth-provider-dev.yml
Group: DEFAULT_GROUP
配置格式: YAML
```

**配置内容：**

```yaml
server:
  port: 8081

spring:
  application:
    name: duda-auth-provider

  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://120.26.170.213:3306/duda_auth?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: duda2024
    druid:
      initial-size: 5
      min-idle: 5
      max-active: 20
```

### 第三步：启动服务（IDEA）

#### 方式 1: 使用环境变量（推荐）

1. 在 IDEA 中打开 `duda-auth-provider` 服务
2. 找到主启动类（如 `AuthProviderApplication.java`）
3. 右键 → `Modify Run Configuration...`
4. 添加环境变量：

```
SPRING_PROFILES_ACTIVE=dev
NACOS_USERNAME=nacos
NACOS_PASSWORD=nacos
NACOS_NAMESPACE=duda-dev
```

5. 点击 `Run` 按钮

#### 方式 2: 使用 .env 文件

如果配置了 `.env` 文件，直接运行即可：

```bash
# 在项目根目录
source .env

# 启动服务（IDEA 中运行）
```

## 🔍 故障排查

### 问题 1: 服务无法启动

**症状：** 启动失败，日志中有异常

**排查步骤：**

1. 检查 Nacos 配置是否存在
   ```bash
   curl -u nacos:nacos \
     "http://120.26.170.213:8848/nacos/v1/cs/configs?dataId=duda-auth-provider-dev.yml&group=DEFAULT_GROUP&tenant=duda-dev"
   ```

2. 检查数据库是否存在
   ```bash
   mysql -h 120.26.170.213 -u root -pduda2024 -e "SHOW DATABASES LIKE 'duda_auth';"
   ```

3. 检查端口是否被占用
   ```bash
   lsof -i:8081
   ```

### 问题 2: 服务启动但无法注册到 Nacos

**症状：** 服务启动正常，但 Nacos 服务列表中看不到

**排查步骤：**

1. 检查 bootstrap.yml 配置
   ```yaml
   spring:
     cloud:
       nacos:
         discovery:
           server-addr: 120.26.170.213:8848
           namespace: duda-dev  # 必须与配置中心一致
   ```

2. 检查日志中的注册信息
   ```log
   # 查找关键字
   grep "is registered" logs/spring.log
   ```

3. 检查 Nacos 命名空间
   - 确认服务注册的命名空间与查看的命名空间一致

### 问题 3: 配置无法刷新

**症状：** 修改 Nacos 配置后，服务没有更新

**排查步骤：**

1. 检查 `@RefreshScope` 注解
   ```java
   @RefreshScope
   @Configuration
   public class AuthConfig {
       @Value("${duda.auth.jwt.secret}")
       private String jwtSecret;
   }
   ```

2. 检查 shared-configs 配置
   ```yaml
   shared-configs:
     - data-id: common-dev.yml
       refresh: true  # ← 必须为 true
   ```

3. 手动触发刷新
   ```bash
   curl -X POST http://localhost:8081/actuator/refresh
   ```
