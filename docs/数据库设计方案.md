## 二、数据库表设计

### 主表：users（用户中台核心表）

**设计原则**：只存储用户的核心身份信息和必填字段

```sql
CREATE TABLE users (
    -- 主键
    id BIGINT PRIMARY KEY COMMENT '用户ID（雪花算法）',

    -- 核心身份信息（必填）
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '密码（BCrypt加密）',
    phone VARCHAR(20) UNIQUE COMMENT '手机号',
    email VARCHAR(100) UNIQUE COMMENT '邮箱',
    avatar VARCHAR(500) COMMENT '头像URL',

    -- 用户类型和状态
    user_type VARCHAR(20) NOT NULL DEFAULT 'normal' COMMENT '用户类型：normal-普通用户,merchant-商家,operator-运营,admin-管理员',
    status VARCHAR(20) NOT NULL DEFAULT 'inactive' COMMENT '状态：active-激活, inactive-未激活, suspended-暂停',

    -- 基础资料（可选）
    real_name VARCHAR(50) COMMENT '真实姓名',

    -- 登录信息
    last_login_time DATETIME COMMENT '最后登录时间',
    last_login_ip VARCHAR(50) COMMENT '最后登录IP',

    -- 系统字段
    remark VARCHAR(500) COMMENT '备注',
    deleted TINYINT DEFAULT 0 COMMENT '逻辑删除：0-未删除, 1-已删除',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    create_by VARCHAR(50) COMMENT '创建人',
    update_by VARCHAR(50) COMMENT '更新人',

    -- 索引
    INDEX idx_phone (phone),
    INDEX idx_email (email),
    INDEX idx_user_type (user_type),
    INDEX idx_status (status),
    INDEX idx_create_time (create_time)
) COMMENT='用户中台核心表（分100张表）';
```

**字段数量**：20 个（包含系统字段）

### 关联表1：user_profiles（用户扩展资料表）

**设计原则**：存储用户的扩展信息，不是每个用户都有
