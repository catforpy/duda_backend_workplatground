# 如何连接到服务器基础设施

## 🚀 快速开始

### 1. 复制环境变量模板

```bash
cd /Volumes/DudaDate/DudaNexus

# 复制环境变量模板
cp .env.template .env

# 编辑配置，修改密码
vim .env
```

### 2. 测试服务器连接

```bash
# 运行连接测试脚本
./scripts/test-connection.sh
```

**预期输出：**
```
1️⃣  测试 Nacos...
   ✅ Nacos 连接成功
   📍 控制台: http://120.26.170.213:8848/nacos
   👤 账号: nacos / nacos

2️⃣  测试 Redis...
   ✅ Redis 连接成功

3️⃣  测试 MySQL...
   ✅ MySQL 端口可访问
   📍 主机: 120.26.170.213
   📍 端口: 3306
   👤 用户: root
```

### 3. 配置服务连接

#### 方法一：使用 bootstrap.yml

在 `src/main/resources/bootstrap.yml` 中配置：

```yaml
spring:
  application:
    name: your-service-name
  cloud:
    nacos:
      username: nacos        # 🔐 认证必需
      password: nacos        # 🔐 认证必需
      discovery:
        server-addr: 120.26.170.213:8848
        namespace: public
        group: DEFAULT_GROUP
      config:
        server-addr: 120.26.170.213:8848
        namespace: public
        group: DEFAULT_GROUP
```

#### 方法二：使用环境变量

在 IDEA 中配置环境变量：

```
Run → Edit Configurations
  → Environment
    → NACOS_HOST=120.26.170.213
    → NACOS_PORT=8848
    → DB_PASSWORD=your_password
```

## 🔗 具体配置步骤

### 步骤 1：配置 Nacos

1. **访问 Nacos 控制台**
   ```
   http://120.26.170.213:8848/nacos
   ```

2. **登录**
   ```
   用户名: nacos
   密码: nacos
   ```

3. **创建命名空间**
   - 命名空间 ID: `duda-dev`
   - 命名空间名称: `都达开发环境`
   - 描述: DudaNexus 开发环境配置

4. **导入配置（可选）**
   - 在 `duda-dev` 命名空间下导入配置
   - 配置文件格式: YAML
   - Data ID: `your-service-dev.yml`

### 步骤 2：配置数据库

#### 创建数据库

```sql
-- 连接 MySQL
mysql -h 120.26.170.213 -P 3306 -u root -p

-- 创建数据库
CREATE DATABASE IF NOT EXISTS duda_dev
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE duda_dev;
```

#### 导入表结构（如果有）

```bash
mysql -h 120.26.170.213 -P 3306 -u root -p duda_dev < init-sql/duda_user.sql
```

### 步骤 3：配置 Redis

```bash
# 连接 Redis
redis-cli -h 120.26.170.213 -p 6379 -a duda2024

# 测试
ping

# 查看信息
info server
```

## 📝 配置文件模板

### bootstrap.yml（推荐）

```yaml
spring:
  application:
    name: ${SPRING_APPLICATION_NAME:duda-user-provider}
  cloud:
    nacos:
      username: ${NACOS_USERNAME:nacos}
      password: ${NACOS_PASSWORD:nacos}
      discovery:
        server-addr: ${NACOS_HOST:120.26.170.213}:${NACOS_PORT:8848}
        namespace: ${NACOS_NAMESPACE:public}
        group: ${NACOS_GROUP:DEFAULT_GROUP}
      config:
        server-addr: ${NACOS_HOST:120.26.170.213}:${NACOS_PORT:8848}
        namespace: ${NACOS_NAMESPACE:public}
        group: ${NACOS_GROUP:DEFAULT_GROUP}
        file-extension: yml
```

### application.yml（本地开发）

```yaml
server:
  port: ${SERVER_PORT:8082}

spring:
  profiles:
    active: dev

  datasource:
    url: jdbc:mysql://${DB_HOST:120.26.170.213}:${DB_PORT:3306}/duda_dev?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai&useSSL=false
    username: ${DB_USER:root}
    password: ${DB_PASSWORD:duda2024}

  data:
    redis:
      host: ${REDIS_HOST:120.26.170.213}
      port: ${REDIS_PORT:6379}
      password: ${REDIS_PASSWORD:duda2024}
      database: 0
```

## ⚠️ 注意事项

### 安全

1. **当前认证状态**
   - ✅ Nacos: 已开启认证（nacos/nacos）
   - ✅ Redis: 已配置密码（duda2024）
   - ✅ MySQL: 已配置密码（root/duda2024）

2. **环境变量管理**
   - 使用 `.env` 文件管理本地环境变量
   - 生产环境使用独立的密钥管理
   - 不要将 `.env` 提交到 Git

3. **网络访问**
   - 当前服务器在公网
   - 建议配置防火墙规则
   - 限制访问来源 IP

### 性能优化

1. **RocketMQ Broker 内存优化**
   - 当前使用 2.01GB
   - 建议调整 JVM 堆内存为 1GB
   - 参考文档：`docs/服务器基础设施配置.md`

2. **MySQL 连接池优化**
   ```yaml
   druid:
     initial-size: 5
     min-idle: 5
     max-active: 20
   ```

## 🔍 故障排查

### Nacos 连接失败

```bash
# 检查 Nacos 是否运行（需要认证）
curl -u nacos:nacos http://120.26.170.213:8848/nacos/v1/console/health/readiness

# 或访问控制台
open http://120.26.170.213:8848/nacos
```

### MySQL 连接失败

```bash
# 检查端口是否开放
nc -zv 120.26.170.213 3306

# 测试连接
mysql -h 120.26.170.213 -P 3306 -u root -pduda2024 -e "SELECT 1"
```

```bash
# 检查端口
nc -zv 120.26.170.213 6379

```
