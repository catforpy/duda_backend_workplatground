## 🎯 核心原则

### ⚠️ 第一原则：前端永远不存储永久秘钥

```java
❌ 错误示例：

✅ 正确示例：

```

```yaml
  jwt:
```

## 📐 秘钥生成规范

### 1. RSA密钥对（2048位）

#### 后端生成

```java
package com.duda.common.security.key;

import java.security.KeyPair;
import java.security.KeyPairGenerator;
import java.security.SecureRandom;
import java.util.Base64;

/**
 * RSA密钥对生成器
 */
public class RSAKeyGenerator {

    private static final int KEY_SIZE = 2048;

    /**
     * 生成RSA密钥对
     */
    public static KeyPair generateKeyPair() throws Exception {
        KeyPairGenerator keyPairGenerator = KeyPairGenerator.getInstance("RSA");
        keyPairGenerator.initialize(KEY_SIZE, new SecureRandom());
        return keyPairGenerator.generateKeyPair();
    }

    /**
     * 将公钥转换为Base64编码（用于前端）
     */
    public static String publicKeyToBase64(PublicKey publicKey) {
        return Base64.getEncoder().encodeToString(publicKey.getEncoded());
    }

    /**
     * 将私钥转换为Base64编码（用于配置存储）
     */
    public static String privateKeyToBase64(PrivateKey privateKey) {
        return Base64.getEncoder().encodeToString(privateKey.getEncoded());
    }
}
```

#### 配置存储

```yaml
# Nacos配置中心
encryption:
  rsa:
    # 私钥（加密存储，仅后端可访问）
    private-key: ENC(AES256加密后的Base64私钥)
    # 公钥（明文存储，前端可获取）
    public-key-base64: MFwwDQYJKoZIhvcNAQEBBQADSwAwSAJBAKX2Q7...
```

#### 前端获取

```javascript
// 前端通过接口获取RSA公钥
async function getPublicKey() {
  const response = await fetch('/api/encryption/public-key');
  const data = await response.json();

  if (data.success) {
    // 保存公钥（只读，不可修改）
    window.RSA_PUBLIC_KEY = data.data;
  }
}
```

### 2. AES临时密钥（256位）

#### 前端生成（每次请求）

```javascript
import CryptoJS from 'crypto-js';

/**
 * 生成临时AES密钥（256位）
 */
function generateAESKey() {
  return CryptoJS.lib.WordArray.random(16).toString();
}

/**
 * 使用AES密钥加密数据
 */
function encryptWithAES(data, aesKey) {
  const key = CryptoJS.enc.Utf8.parse(aesKey);
  const iv = CryptoJS.lib.WordArray.random(16); // 随机IV

  const encrypted = CryptoJS.AES.encrypt(data, key, {
    iv: iv,
    mode: CryptoJS.mode.CBC,
    padding: CryptoJS.pad.Pkcs7
  });

  // 返回 IV + 密文
  return iv.toString() + ':' + encrypted.ciphertext.toString(CryptoJS.enc.Base64);
}
```

#### 后端解密

```java
package com.duda.common.security.crypto;

import javax.crypto.Cipher;
import javax.crypto.spec.IvParameterSpec;
import javax.crypto.spec.SecretKeySpec;
import java.util.Base64;

/**
 * AES加密工具
 */
public class AESUtil {

    private static final String ALGORITHM = "AES/CBC/PKCS5Padding";
    private static final int KEY_SIZE = 256;
    private static final int IV_SIZE = 16;

    /**
     * AES解密
     * @param encryptedData 加密数据（Base64）
     * @param key AES密钥
     * @return 解密后的明文
     */
    public static String decrypt(String encryptedData, String key) throws Exception {
        // 分离IV和密文
        String[] parts = encryptedData.split(":");
        byte[] iv = Base64.getDecoder().decode(parts[0]);
        byte[] cipherText = Base64.getDecoder().decode(parts[1]);

        // 创建密钥和IV
        SecretKeySpec secretKey = new SecretKeySpec(key.getBytes(), "AES");
        IvParameterSpec ivSpec = new IvParameterSpec(iv);

        // 解密
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        cipher.init(Cipher.DECRYPT_MODE, secretKey, ivSpec);
        byte[] decrypted = cipher.doFinal(cipherText);

        return new String(decrypted);
    }
}
```

### 3. JWT签名密钥（HS256/RS256）

#### 后端生成

```java
package com.duda.common.security.jwt;

import io.jsonwebtoken.Jwts;
import io.jsonwebtoken.SignatureAlgorithm;
import io.jsonwebtoken.security.Keys;

import javax.crypto.SecretKey;
import java.util.Base64;

/**
 * JWT密钥生成器
 */
public class JWTKeyGenerator {

    /**
     * 生成HS256密钥（对称加密）
     */
    public static String generateHS256Key() {
        SecretKey key = Keys.secretKeyFor(SignatureAlgorithm.HS256);
        return Base64.getEncoder().encodeToString(key.getEncoded());
    }

    /**
     * 生成RS256密钥对（非对称加密）
     */
    public static KeyPair generateRS256KeyPair() {
        return Keys.keyPairFor(SignatureAlgorithm.RS256);
    }
}
```

#### 配置存储

```yaml
# Nacos配置中心

  private-key: ${JWT_RSA_PRIVATE_KEY}
  public-key: ${JWT_RSA_PUBLIC_KEY}
```

## 🔄 密钥轮换机制

### 轮换策略

| 密钥类型 | 轮换周期 | 轮换方式 | 影响范围 |
|---------|---------|---------|---------|
| **AES临时密钥** | 每次请求 | 自动轮换 | 单次请求 |
| **会话密钥** | 1小时 | 自动轮换 | 当前会话 |
| **JWT签名密钥** | 每季度 | 手动轮换 | 所有Token |
| **数据库密码** | 每月 | 手动轮换 | 数据库连接 |
| **支付密钥** | 每月 | 手动轮换 | 支付功能 |
| **RSA密钥对** | 每年 | 手动轮换 | 所有加密请求 |

### 轮换实施步骤

#### 步骤1：准备新密钥

```java
/**
 * 密钥轮换服务
 */
@Service
public class KeyRotationService {

    @Autowired
    private KeyManagementService keyManagementService;

    /**
     * 生成新密钥
     */
    public void rotateKey(String keyId) {
        // 1. 生成新版本的密钥
        String newKey = generateNewKey();

        // 2. 保存到密钥管理系统（新版本）
        keyManagementService.saveNewVersion(keyId, newKey);

        // 3. 标记为活跃状态
        keyManagementService.activateVersion(keyId, newVersionId);

        // 4. 旧密钥保留一段时间（用于解密旧数据）
        keyManagementService.deprecateVersion(keyId, oldVersionId);
    }
}
```

#### 步骤2：平滑过渡

```yaml
# 密钥版本配置
encryption:
  rsa:
    # 主密钥（当前使用）
    private-key-v2: ${RSA_PRIVATE_KEY_V2}
    # 旧密钥（保留30天，用于解密旧数据）
    private-key-v1: ${RSA_PRIVATE_KEY_V1}
    active-version: v2
```

#### 步骤3：验证

```java
/**
 * 密钥轮换验证
 */
@Test
public void testKeyRotation() {
    // 1. 用新密钥加密
    String encrypted = encryptWithNewKey("data");

    // 2. 用新密钥解密（成功）
    String decrypted1 = decryptWithNewKey(encrypted);
    assertEquals("data", decrypted1);

    // 3. 用旧密钥解密旧数据（成功）
    String oldEncrypted = "old_encrypted_data";
    String decrypted2 = decryptWithOldKey(oldEncrypted);
    assertEquals("old_data", decrypted2);

    // 4. 旧密钥解密新数据（失败）
    assertThrows(Exception.class, () -> {
        decryptWithOldKey(encrypted);
    });
}
```

## 🔑 前端密钥管理

### 临时密钥生成

```javascript
/**
 * 前端密钥管理
 */
class KeyManager {
  constructor() {
    this.sessionKeys = new Map(); // 会话密钥缓存
    this.rsaPublicKey = null;     // RSA公钥
  }

  /**
   * 初始化：获取RSA公钥
   */
  async init() {
    const response = await fetch('/api/encryption/public-key');
    const data = await response.json();
    this.rsaPublicKey = data.data;
  }

  /**
   * 生成临时AES密钥
   */
  generateAESKey() {
    const key = CryptoJS.lib.WordArray.random(16).toString();
    const keyId = Date.now() + '_' + Math.random().toString(36);
    this.sessionKeys.set(keyId, {
      key: key,
      createdAt: Date.now(),
      expiresAt: Date.now() + 3600000 // 1小时后过期
    });
    return { keyId, key };
  }

  /**
   * 清理过期密钥
   */
  cleanExpiredKeys() {
    const now = Date.now();
    for (const [keyId, keyData] of this.sessionKeys.entries()) {
      if (now > keyData.expiresAt) {
        this.sessionKeys.delete(keyId);
      }
    }
  }

  /**
   * 用RSA公钥加密AES密钥
   */
  encryptAESKeyWithRSA(aesKey) {
    const encrypt = new JSEncrypt();
    encrypt.setPublicKey(this.rsaPublicKey);
    return encrypt.encrypt(aesKey);
  }
}

// 使用示例
const keyManager = new KeyManager();
await keyManager.init();

// 每次请求生成新密钥
const { keyId, key } = keyManager.generateAESKey();
const encryptedAESKey = keyManager.encryptAESKeyWithRSA(key);
```

## 🔍 密钥安全审计

### 审计内容

```java
/**
 * 密钥使用审计
 */
@Aspect
@Component
public class KeyUsageAuditAspect {

    @Autowired
    private AuditLogService auditLogService;

    /**
     * 记录密钥访问
     */
    @Around("@annotation(UseKey)")
    public Object auditKeyUsage(ProceedingJoinPoint pjp) throws Throwable {
        String keyId = getKeyIdFromAnnotation(pjp);

        // 记录访问日志
        auditLogService.log(AuditLog.builder()
            .action("KEY_ACCESS")
            .resource(keyId)
            .userId(getCurrentUserId())
            .timestamp(System.currentTimeMillis())
            .success(true)
            .build());

        return pjp.proceed();
    }
}
```

## 🚨 应急响应

### 密钥泄露处理流程

#### 1. 确认泄露

```bash
# 检查Git历史
git log --all --full-history --source -- "*key*"

# 检查日志文件
grep -r "private.key" /var/log/

# 检查代码仓库
git log --all --source -- "*password*"
```

#### 2. 立即冻结

```java
/**
 * 冻结泄露密钥
 */
public void freezeLeakedKey(String keyId) {
    // 1. 标记密钥为冻结状态
    keyManagementService.freezeKey(keyId);

    // 2. 通知所有服务停止使用该密钥
    notificationService.alertAllServices("KEY_FROZEN", keyId);

    // 3. 记录安全事件
    securityEventService.record(SecurityEvent.KEY_LEAK, keyId);
}
```

#### 3. 紧急轮换

```java
/**
 * 紧急密钥轮换
 */
public void emergencyKeyRotation(String keyId) {
    // 1. 立即生成新密钥
    String newKey = generateNewKey();

    // 2. 激活新密钥
    keyManagementService.activateKey(keyId, newKey);

    // 3. 重启所有受影响的服务
    restartAffectedServices();

    // 4. 验证新密钥生效
    verifyNewKeyActive();
}
```

#### 4. 影响评估

```java
/**
 * 评估密钥泄露影响
 */
public SecurityIncident assessImpact(String keyId) {
    // 1. 查询密钥使用历史
    List<KeyUsageLog> logs = auditLogService.getUsageLogs(keyId);

    // 2. 分析泄露时间窗口
    Date leakTime = detectLeakTime(logs);

    // 3. 分析影响的业务
    List<String> affectedServices = getAffectedServices(keyId);

    // 4. 评估数据泄露风险
    RiskLevel riskLevel = assessDataLeakRisk(keyId, leakTime);

    return SecurityIncident.builder()
        .keyId(keyId)
        .leakTime(leakTime)
        .affectedServices(affectedServices)
        .riskLevel(riskLevel)
        .build();
}
```
