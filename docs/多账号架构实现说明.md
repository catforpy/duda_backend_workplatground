## 📋 功能概述

实现了用户多账号登录系统,支持一个用户通过多种方式登录,并为后续的**账号合并功能**打下基础。

### 核心特性

- ✅ 支持多种登录方式: 用户名、手机号、邮箱、微信、QQ、支付宝、微博
- ✅ 一个用户可以有多个登录方式
- ✅ 为实名制账号合并打下基础
- ✅ 向后兼容原有数据结构
- ✅ 密码 BCrypt 加密存储

### 1. UserAccount 实体类

```java
@Data

@Schema(description = "用户登录账号实体")
public class UserAccount {

    @TableId(type = IdType.AUTO)
    private Long id;

    @Schema(description = "关联的用户ID")
    private Long userId;

    @Schema(description = "登录方式: username, phone, email, wechat, qq, alipay, weibo")
    private String loginType;

    @Schema(description = "登录账号")
    private String loginAccount;

    @Schema(description = "密码(仅username和email类型有)")
    private String password;

    @Schema(description = "是否已验证")
    private Boolean verified;

    @Schema(description = "是否为主账号")
    private Boolean isPrimary;

    private String status;
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
}
```

### 2. UserAccountMapper 接口

**文件**: `duda-user-provider/src/main/java/com/duda/user/mapper/UserAccountMapper.java`

```java
@Mapper
public interface UserAccountMapper extends BaseMapper<UserAccount> {

    /**
     * 根据登录方式和登录账号查询
     */
    @Select("SELECT * FROM user_accounts WHERE login_type = #{loginType} AND login_account = #{loginAccount} AND status = 'active' LIMIT 1")
    UserAccount selectByLoginTypeAndAccount(
        @Param("loginType") String loginType,
        @Param("loginAccount") String loginAccount
    );

    /**
     * 检查登录账号是否已存在
     */
    @Select("SELECT COUNT(*) > 0 FROM user_accounts WHERE login_type = #{loginType} AND login_account = #{loginAccount}")
    boolean existsByLoginTypeAndAccount(
        @Param("loginType") String loginType,
        @Param("loginAccount") String loginAccount
    );
}
```

### 3. 用户注册逻辑

**文件**: `duda-user-provider/src/main/java/com/duda/user/service/impl/UserServiceImpl.java`

**核心逻辑**:
```java
@Override
@Transactional(rollbackFor = Exception.class)
public Long register(UserRegisterReqDTO registerReq) {
    // 1. 检查用户名是否已存在
    boolean usernameExists = userAccountMapper.existsByLoginTypeAndAccount(
        "username", registerReq.getUsername()
    );
    if (usernameExists) {
        throw new BizException("用户名已存在");
    }

    // 2. 生成密码hash
    String hashedPassword = BCrypt.hashpw(registerReq.getPassword());

    // 3. 创建用户(Users表)
    Long userId = idGeneratorRpc.generateUserId();
    UserPO userPO = new UserPO();
    userPO.setId(userId);
    userPO.setUsername(registerReq.getUsername()); // 兼容旧字段
    userPO.setPassword(hashedPassword);          // 兼容旧字段
    userPO.setRealName(registerReq.getRealName());
    userPO.setStatus("inactive");

    // 4. 创建登录账号(user_accounts表)
    UserAccount userAccount = new UserAccount();
    userAccount.setUserId(userId);
    userAccount.setLoginType("username");
    userAccount.setLoginAccount(registerReq.getUsername());
    userAccount.setPassword(hashedPassword);  // 使用相同的密码hash
    userAccount.setVerified(true);
    userAccount.setIsPrimary(true);
    userAccount.setStatus("active");

    // 5. 保存到数据库
    userMapper.insert(userPO);
    userAccountMapper.insert(userAccount);

    return userId;
}
```

### 4. 用户登录逻辑

```java
@Override
public UserDTO login(UserLoginReqDTO loginReq) {
    // 1. 从 user_accounts 表查询登录账号
    UserAccount userAccount = userAccountMapper.selectByLoginTypeAndAccount(
        "username", loginReq.getUsername()
    );

    if (userAccount == null) {
        throw new BizException("用户名或密码错误");
    }

    // 2. 根据 user_id 查询用户信息
    LambdaQueryWrapper<UserPO> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(UserPO::getId, userAccount.getUserId());
    UserPO userPO = userMapper.selectOne(wrapper);

    if (userPO == null) {
        throw new BizException("用户信息不存在");
    }

    // 3. 验证密码(从 user_account 表)
    boolean passwordMatch = BCrypt.checkpw(
        loginReq.getPassword(),
        userAccount.getPassword()
    );

    if (!passwordMatch) {
        throw new BizException("用户名或密码错误");
    }

    // 4. 更新最后登录时间
    userPO.setLastLoginTime(LocalDateTime.now());
    userMapper.updateById(userPO);

    // 5. 首次登录自动激活
    if ("inactive".equals(userPO.getStatus())) {
        userPO.setStatus("active");
        userMapper.updateById(userPO);
    }

    return BeanCopyUtils.copy(userPO, UserDTO.class);
}
```

## 🎯 设计优势

### 1. 扩展性强
- 轻松添加新的登录方式,只需修改 `login_type` 枚举值
- 无需修改核心表结构

### 2. 安全性高
- 密码使用 BCrypt 加密存储
- 每种登录方式可以独立管理验证状态

### 3. 向后兼容
- 保留了 `users` 表的 `username` 和 `password` 字段
- 已有数据无需强制迁移,可以平滑过渡

### 4. 账号合并基础
- 通过 `user_id` 关联,一个用户可以有多个登录方式
- 为实名制场景下的账号合并打下基础

## 🚀 后续功能规划

### 1. 账号合并功能 (实名制场景)

**场景描述**:
- 用户A先用手机号注册 → 获得 userId_1
- 同一用户用微信注册 → 获得 userId_2
- 实名认证时,系统识别为同一人
- 将两个账号合并为一个主账号

**实现方案**:
```sql
-- 1. 选择一个主账号
UPDATE user_accounts
SET user_id = userId_1  -- 统一指向主账号
WHERE user_id IN (userId_1, userId_2);

-- 2. 标记原账号为已合并
UPDATE users SET deleted = 1 WHERE id = userId_2;
```

### 2. 第三方登录

- 微信登录 (login_type = 'wechat')
- QQ登录 (login_type = 'qq')
- 支付宝登录 (login_type = 'alipay')
- 微博登录 (login_type = 'weibo')

### 3. 手机验证码登录

- 无需密码
- 验证码验证后直接登录
- login_type = 'phone'

### 4. 多账号管理

- 用户查看自己绑定的所有登录方式
- 解绑某个登录方式
- 添加新的登录方式

```sql
CREATE TABLE user_accounts (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    user_shard INT DEFAULT 0,
    login_type VARCHAR(20) NOT NULL,
    login_account VARCHAR(100) NOT NULL,
    password VARCHAR(100),
    verified BOOLEAN DEFAULT FALSE,
    is_primary BOOLEAN DEFAULT FALSE,
    status VARCHAR(20) DEFAULT 'active',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_login_type_account (login_type, login_account),
    INDEX idx_user_id (user_id),
    INDEX idx_login_type (login_type)
) COMMENT='用户登录账号表';

-- 2. 迁移现有数据
INSERT INTO user_accounts (user_id, login_type, login_account, password, verified, is_primary)
SELECT
    id,
    'username',
    username,
    password,
    TRUE,
    TRUE
FROM users
WHERE username IS NOT NULL AND deleted = 0;
```
