# 密钥管理配置指南

## 📋 概述

本系统使用 **数据库 + AES-256-GCM 加密** 方案存储和管理云存储服务（阿里云OSS、腾讯云COS等）的访问密钥。

### 工作流程

```
用户注册云存储 → AES加密密钥 → 存入数据库 → 使用时解密 → 调用OSS API
```

### 安全特性

- ✅ **AES-256-GCM 加密** - 业界标准加密算法
- ✅ **每用户独立密钥** - 每个客户的云存储密钥独立加密存储
- ✅ **随机IV** - 每次加密使用不同的初始化向量
- ✅ **认证加密** - GCM模式提供数据完整性和真实性验证
- ✅ **密钥与代码分离** - 加密密钥存储在配置文件/环境变量中

## 🔧 配置步骤

### 第一步：配置AES加密密钥

在 `application.yml` 或 `application.properties` 中配置AES密钥：

```yaml
# application.yml
encryption:
  aes:
    # ⚠️ 重要：请修改为32字节长的随机字符串（生产环境必须使用环境变量）
    key: ChangeThisKeyTo32BytesLong!!
```

或者：

```properties
# application.properties
encryption.aes.key=ChangeThisKeyTo32BytesLong!!
```

### 第二步：生成32字节随机密钥

**方法1：使用Java代码**

```java
import java.security.SecureRandom;
import java.util.Base64;

public class GenerateKey {
    public static void main(String[] args) {
        byte[] key = new byte[32];
        new SecureRandom().nextBytes(key);
        String base64Key = Base64.getEncoder().encodeToString(key);
        System.out.println("AES密钥（Base64）: " + base64Key);
    }
}
```

**方法2：使用OpenSSL**

```bash
openssl rand -base64 32
```

**方法3：使用Python**

```python
import secrets
import base64
key = secrets.token_bytes(32)
print(base64.b64encode(key).decode())
```

### 第三步：配置环境变量（生产环境推荐）

```bash
# Linux/Mac
export ENCRYPTION_AES_KEY="your-32-byte-random-key-here"

# Windows
set ENCRYPTION_AES_KEY=your-32-byte-random-key-here
```

在 `application.yml` 中引用环境变量：

```yaml
encryption:
  aes:
    key: ${ENCRYPTION_AES_KEY:ChangeThisKeyTo32BytesLong!!}
```

## 📊 数据库配置

### 存储Bucket时加密密钥

当用户创建或更新Bucket配置时，系统会自动加密密钥：

```java
// 示例：创建Bucket配置
BucketConfig bucketConfig = BucketConfig.builder()
    .bucketName("my-bucket")
    .storageType("aliyun-oss")
    .region("cn-hangzhou")
    .accessKeyId(aesEncryptUtil.encrypt("LTAI5tXXX..."))  // 自动加密
    .accessKeySecret(aesEncryptUtil.encrypt("XXXsecret..."))  // 自动加密
    .stsRoleArn("acs:ram::123456789:role/oss-sts-role")
    .build();

bucketConfigMapper.insert(bucketConfig);
```

### 数据库中的密文格式

```
access_key_id: "YWJjZGVmZ2hpamsxMjM0NTY3YWJj="  (Base64编码的密文)
access_key_secret: "eHl6YWJjMTIzNDU2Nzg5MDEyMzQ1Njc4OWFiYw=="  (Base64编码的密文)
```

**结构**：`IV(12字节) + 密文 + 认证标签(16字节)` → Base64编码

## 🔐 使用示例

### 示例1：用户创建Bucket配置

```java
@Service
public class BucketServiceImpl {

    @Autowired
    private BucketConfigMapper bucketConfigMapper;

    @Autowired
    private AesEncryptUtil aesEncryptUtil;

    public void createBucket(CreateBucketReqDTO req) {
        // 1. 加密用户提交的密钥
        String encryptedKeyId = aesEncryptUtil.encrypt(req.getAccessKeyId());
        String encryptedKeySecret = aesEncryptUtil.encrypt(req.getAccessKeySecret());

        // 2. 保存到数据库
        BucketConfig config = BucketConfig.builder()
            .bucketName(req.getBucketName())
            .storageType(req.getStorageType())
            .region(req.getRegion())
            .accessKeyId(encryptedKeyId)  // 加密后的密钥
            .accessKeySecret(encryptedKeySecret)  // 加密后的密钥
            .stsRoleArn(req.getStsRoleArn())
            .userId(req.getUserId())
            .status("ACTIVE")
            .build();

        bucketConfigMapper.insert(config);
    }
}
```

### 示例2：STS服务自动解密密钥

```java
@Service
public class STSService {

    @Autowired
    private BucketConfigMapper bucketConfigMapper;

    @Autowired
    private AesEncryptUtil aesEncryptUtil;

    public STSCredentialsDTO generateSTSCredentials(String bucketName, String prefix, Long duration) {
        // 1. 从数据库查询Bucket配置（密钥是加密的）
        BucketConfig config = bucketConfigMapper.selectByBucketName(bucketName);

        // 2. 解密密钥
        String accessKeyId = aesEncryptUtil.decrypt(config.getAccessKeyId());
        String accessKeySecret = aesEncryptUtil.decrypt(config.getAccessKeySecret());

        // 3. 使用解密后的密钥调用阿里云STS API
        DefaultProfile profile = DefaultProfile.getProfile(
            config.getRegion(),
            accessKeyId,  // 解密后的明文密钥
            accessKeySecret  // 解密后的明文密钥
        );

        // ... 后续STS调用逻辑
    }
}
```

## ⚠️ 安全注意事项

### 1. 保护AES主密钥

- ❌ **不要将AES主密钥硬编码在代码中**
- ❌ **不要将AES主密钥提交到Git仓库**
- ✅ **使用环境变量或配置中心**
- ✅ **生产环境使用KMS（密钥管理服务）**

### 2. 密钥轮换

建议每3-6个月轮换一次AES主密钥：

```java
// 密钥轮换脚本
public void rotateEncryptionKey() {
    // 1. 获取所有Bucket配置
    List<BucketConfig> configs = bucketConfigMapper.selectAll();

    // 2. 使用旧密钥解密，新密钥加密
    for (BucketConfig config : configs) {
        String decryptedKeyId = oldAesUtil.decrypt(config.getAccessKeyId());
        String decryptedSecret = oldAesUtil.decrypt(config.getAccessKeySecret());

        config.setAccessKeyId(newAesUtil.encrypt(decryptedKeyId));
        config.setAccessKeySecret(newAesUtil.encrypt(decryptedSecret));

        bucketConfigMapper.update(config);
    }
}
```

### 3. 数据库备份加密

- ✅ 使用透明数据加密（TDE）
- ✅ 定期备份数据库
- ✅ 备份文件也要加密存储

## 📈 升级到KMS

### 当前阶段：数据库+AES（开发/测试）

- 成本：低
- 安全性：中等
- 适用场景：内部系统、开发测试

### 生产环境：数据库+KMS（推荐）

- 成本：约¥60/月（10万次调用）
- 安全性：高
- 适用场景：生产环境、客户数据

#### 升级步骤（无缝切换）

1. **引入KMS SDK**

```xml
<dependency>
    <groupId>com.aliyun</groupId>
    <artifactId>aliyun-java-sdk-kms</artifactId>
    <version>2.x.x</version>
</dependency>
```

2. **修改加密工具类**

```java
@Service
public class KMSService {

    @Value("${kms.provider:aliyun}")  // aliyun/tencent/aws
    private String kmsProvider;

    public String encrypt(String plaintext) {
        if ("aliyun".equals(kmsProvider)) {
            // 调用阿里云KMS加密
            return aliyunKMS.encrypt(plaintext);
        } else {
            // 降级使用AES
            return aesEncryptUtil.encrypt(plaintext);
        }
    }

    public String decrypt(String ciphertext) {
        if ("aliyun".equals(kmsProvider)) {
            // 调用阿里云KMS解密
            return aliyunKMS.decrypt(ciphertext);
        } else {
            // 降级使用AES
            return aesEncryptUtil.decrypt(ciphertext);
        }
    }
}
```

3. **配置KMS**

```yaml
kms:
  provider: aliyun
  aliyun:
    region: cn-hangzhou
    key-id: your-kms-key-id
```

4. **数据无需迁移**

- 存储在数据库中的密文无需改变
- 只需要使用KMS重新加密一次即可（可选，更安全）

## 📞 常见问题

### Q1: AES密钥长度必须是32字节吗？

A: 是的，AES-256需要32字节（256位）的密钥。如果密钥长度不足，系统会用0填充；如果超过32字节，会截断。

### Q2: 忘记AES主密钥怎么办？

A: **无法恢复**。所有加密的数据都无法解密。建议：
- 将主密钥备份在安全的地方（如密码管理器）
- 使用KMS服务避免单点故障

### Q3: 数据库被攻击，密文会泄露吗？

A: 会泄露，但是：
- 攻击者拿到的是密文，不是明文
- 没有AES主密钥，无法解密
- AES-256-GCM 目前无法暴力破解

### Q4: 性能影响大吗？

A: **几乎无影响**：
- AES-256-GCM 加密速度：约 1GB/s
- 100字节密钥加密耗时：< 0.1ms
- 数据库查询解密总耗时：< 5ms

### Q5: 可以支持腾讯云、七牛云吗？

A: **完全支持**。本系统是云中立的：
- 每个Bucket配置独立的云存储账号
- 通过 `storage_type` 字段区分（aliyun-oss/tencent-cos/qiniu-kodo/minio）
- 密钥统一加密存储
