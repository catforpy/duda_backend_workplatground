## ❌ 删除的表

### 1. file_statistics（文件统计表）

#### 删除原因
- ❌ **功能重复**: bucket_storage_log 已记录存储变化
- ❌ **功能重复**: bucket_traffic_log 已记录流量变化
- ❌ **功能重复**: file_access_log 已记录操作日志
- ❌ **数据可通过聚合查询得到**: 不需要额外维护一张表
- ❌ **增加维护成本**: 需要保证三张表的数据一致性

#### SQL语句
```sql
-- 删除表
DROP TABLE IF EXISTS file_statistics;

-- 说明：
-- 1. 如果历史数据需要保留，先备份
-- 2. 删除后，统计数据通过聚合查询获取
```

#### 替代方案
```sql
-- 如果需要统计某天的文件操作，使用以下查询：

-- 统计新增文件数
SELECT
    DATE(upload_time) as stat_date,
    COUNT(*) as new_files,
    SUM(file_size) as total_size
FROM object_metadata
WHERE upload_time >= '2026-03-01' AND upload_time < '2026-03-02'
    AND status = 'active'
GROUP BY DATE(upload_time);

-- 统计删除文件数
SELECT
    DATE(updated_time) as stat_date,
    COUNT(*) as deleted_files
FROM object_metadata
WHERE updated_time >= '2026-03-01' AND updated_time < '2026-03-02'
    AND status = 'deleted'
GROUP BY DATE(updated_time);

-- 统计上传下载流量
SELECT
    stat_date,
    SUM(upload_traffic) as upload_traffic,
    SUM(download_traffic) as download_traffic,
    SUM(request_count) as request_count
FROM bucket_traffic_log
WHERE stat_date = '2026-03-17'
GROUP BY stat_date;
```

## ➕ 新增的表

### 1. bucket_account（Bucket账户余额表）

#### 创建原因
- ✅ **解决balance字段位置不当问题**: 原来在bucket_billing_config中，但配置表不应存储经常变化的数据
- ✅ **支持复杂账户功能**: 冻结金额、授信额度、累计充值等
- ✅ **记录余额变化历史**: 配合流水表使用
- ✅ **支持并发控制**: 使用version字段实现乐观锁

#### SQL语句
```sql
-- 创建表
CREATE TABLE bucket_account (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    bucket_name VARCHAR(100) NOT NULL UNIQUE COMMENT 'Bucket名称',
    current_balance DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '当前余额（元）',
    frozen_balance DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '冻结金额（元）',
    available_balance DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '可用余额（元=当前余额-冻结余额）',
    total_recharge DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '累计充值（元）',
    total_consumption DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '累计消费（元）',
    credit_line DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '授信额度（元）',
    status VARCHAR(20) NOT NULL DEFAULT 'ACTIVE' COMMENT '账户状态：ACTIVE-正常/FROZEN-冻结/CLOSED-关闭',
    version INT NOT NULL DEFAULT 0 COMMENT '乐观锁版本号',
    created_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    INDEX idx_bucket_name (bucket_name),
    INDEX idx_status (status),
    CONSTRAINT fk_account_bucket FOREIGN KEY (bucket_name) REFERENCES bucket_config(bucket_name) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='Bucket账户余额表';

-- 初始化已有Bucket的账户（从bucket_billing_config迁移balance）
INSERT INTO bucket_account (bucket_name, current_balance, available_balance)
SELECT
    bucket_name,
    balance,
    balance
FROM bucket_billing_config
WHERE balance IS NOT NULL AND balance > 0;
```

#### 字段说明
| 字段名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| id | bigint | 主键ID | 1 |
| bucket_name | varchar(100) | Bucket名称（唯一） | test-bucket |
| current_balance | decimal(10,2) | 当前余额 | 1000.00 |
| frozen_balance | decimal(10,2) | 冻结金额 | 100.00 |
| available_balance | decimal(10,2) | 可用余额 | 900.00 |
| total_recharge | decimal(10,2) | 累计充值 | 5000.00 |
| total_consumption | decimal(10,2) | 累计消费 | 4000.00 |
| credit_line | decimal(10,2) | 授信额度 | 2000.00 |
| status | varchar(20) | 账户状态 | ACTIVE |
| version | int | 乐观锁版本号 | 1 |

#### 使用示例
```sql
-- 充值
UPDATE bucket_account
SET
    current_balance = current_balance + 100.00,
    available_balance = available_balance + 100.00,
    total_recharge = total_recharge + 100.00,
    version = version + 1
WHERE bucket_name = 'test-bucket' AND version = 1;

-- 扣款（使用乐观锁）
UPDATE bucket_account
SET
    current_balance = current_balance - 50.00,
    available_balance = available_balance - 50.00,
    total_consumption = total_consumption + 50.00,
    version = version + 1
WHERE bucket_name = 'test-bucket'
  AND version = 2
  AND available_balance >= 50.00;

-- 冻结金额
UPDATE bucket_account
SET
    frozen_balance = frozen_balance + 100.00,
    available_balance = available_balance - 100.00,
    version = version + 1
WHERE bucket_name = 'test-bucket' AND version = 3;

-- 解冻金额
UPDATE bucket_account
SET
    frozen_balance = frozen_balance - 100.00,
    available_balance = available_balance + 100.00,
    version = version + 1
WHERE bucket_name = 'test-bucket' AND version = 4;
```

## ✏️ 修改的表

### 1. bucket_statistics（Bucket统计表）

#### 修改说明
- 🔴 **删除20+个配置字段**: 这些字段已经在bucket_config表中存在
- ✅ **保留纯统计字段**: 文件数、存储大小、流量、费用等
- ✅ **添加索引**: 优化查询性能

```sql
-- ❌ 当前表中混杂了配置字段（不应该在这里）
acl_type                  -- ACL类型（应该在bucket_config）
```

#### 修改后的表结构
```sql
CREATE TABLE bucket_statistics (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    bucket_name VARCHAR(100) NOT NULL UNIQUE COMMENT 'Bucket名称',
    region VARCHAR(50) COMMENT '区域',
    storage_type VARCHAR(20) COMMENT '存储类型STANDARD/IA/ARCHIVE',

    -- 文件统计
    total_file_count INT DEFAULT 0 COMMENT '总文件数',
    total_storage_size BIGINT DEFAULT 0 COMMENT '总存储空间（字节）',
    image_count INT DEFAULT 0 COMMENT '图片文件数',
    video_count INT DEFAULT 0 COMMENT '视频文件数',
    document_count INT DEFAULT 0 COMMENT '文档文件数',
    other_count INT DEFAULT 0 COMMENT '其他文件数',

    -- 流量统计
    total_traffic_bytes BIGINT DEFAULT 0 COMMENT '总流量（字节）',
    upload_traffic_bytes BIGINT DEFAULT 0 COMMENT '上传流量（字节）',
    download_traffic_bytes BIGINT DEFAULT 0 COMMENT '下载流量（字节）',

    -- 费用统计
    storage_fee DECIMAL(10,2) DEFAULT 0.00 COMMENT '存储费用',
    traffic_fee DECIMAL(10,2) DEFAULT 0.00 COMMENT '流量费用',
    request_fee DECIMAL(10,2) DEFAULT 0.00 COMMENT '请求费用',
    total_fee DECIMAL(10,2) DEFAULT 0.00 COMMENT '总费用',

    -- 时间记录
    last_sync_time DATETIME COMMENT '最后同步时间',
    last_upload_time DATETIME COMMENT '最后上传时间',
    last_download_time DATETIME COMMENT '最后下载时间',
    last_delete_time DATETIME COMMENT '最后删除时间',

    -- 状态
    status VARCHAR(20) DEFAULT 'ACTIVE' COMMENT '状态ACTIVE/SUSPENDED/DELETED',
    created_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    INDEX idx_bucket_name (bucket_name),
    INDEX idx_status (status),
    INDEX idx_updated_time (updated_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='Bucket统计表（仅存储统计数据）';
```

#### 使用示例
```sql
-- 查询Bucket统计 + 配置信息
SELECT
    bs.total_file_count,
    bs.total_storage_size,
    bs.total_fee,
    bc.acl_type,
    bc.cors_enabled,
    bc.domain_name,
    bc.cdn_enabled

LEFT JOIN bucket_config bc ON bs.bucket_name = bc.bucket_name
WHERE bs.bucket_name = 'test-bucket';
```

### 3. upload_record（上传记录表）

#### 修改说明
- ⚠️ **明确upload_id含义**: 改为upload_type + multipart_upload_id
- ⚠️ **添加唯一约束**: 防止重复记录

#### SQL修改语句
```sql
-- ============================================
-- 步骤1: 添加upload_type字段
-- ============================================
ALTER TABLE upload_record
    ADD COLUMN upload_type VARCHAR(20) NOT NULL DEFAULT 'SIMPLE' COMMENT '上传类型：SIMPLE-简单上传/MULTIPART-分片上传/RESUMABLE-断点续传'
    AFTER upload_method;

-- ============================================
-- 步骤2: 重命名upload_id为multipart_upload_id（更明确）
-- ============================================
ALTER TABLE upload_record
    CHANGE COLUMN upload_id multipart_upload_id VARCHAR(255) COMMENT 'OSS分片上传ID（仅分片上传时有值）';

-- ============================================
-- 步骤3: 添加唯一索引（防止重复记录）
-- ============================================
CREATE UNIQUE INDEX uk_bucket_object_time
ON upload_record(bucket_name, object_key, start_time);

-- ============================================
-- 步骤4: 添加组合索引
-- ============================================
CREATE INDEX idx_user_status_time
ON upload_record(user_id, upload_status, start_time);

CREATE INDEX idx_bucket_status_time
ON upload_record(bucket_name, upload_status, start_time);
```

#### 修改后的表结构
```sql
CREATE TABLE upload_record (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    bucket_name VARCHAR(255) NOT NULL COMMENT '存储桶名',
    object_key VARCHAR(1024) NOT NULL COMMENT '对象键',

    -- 上传类型和ID（修改后的字段）
    upload_method VARCHAR(20) NOT NULL COMMENT '上传方式',
    upload_type VARCHAR(20) NOT NULL DEFAULT 'SIMPLE' COMMENT '上传类型：SIMPLE-简单上传/MULTIPART-分片上传/RESUMABLE-断点续传',
    multipart_upload_id VARCHAR(255) COMMENT 'OSS分片上传ID（仅分片上传时有值）',

    -- 用户信息
    user_id BIGINT NOT NULL COMMENT '操作者ID',
    user_shard TINYINT NOT NULL COMMENT '用户ID分片',

    -- 文件信息
    file_name VARCHAR(500) COMMENT '文件名',
    file_size BIGINT NOT NULL COMMENT '文件大小（字节）',
    file_md5 VARCHAR(32) COMMENT '文件MD5',
    content_type VARCHAR(255) COMMENT 'MIME类型',

    -- 分片信息
    part_count INT DEFAULT 0 COMMENT '分片数量',
    part_size BIGINT COMMENT '分片大小',
    uploaded_parts INT DEFAULT 0 COMMENT '已上传分片数',

    -- 状态
    upload_status VARCHAR(20) NOT NULL COMMENT '上传状态',
    start_time DATETIME COMMENT '开始时间',
    complete_time DATETIME COMMENT '完成时间',

    -- 客户端信息
    upload_ip VARCHAR(50) COMMENT '上传IP',
    client_type VARCHAR(20) COMMENT '客户端类型',
    device_id VARCHAR(100) COMMENT '设备ID',
    user_agent VARCHAR(500) COMMENT 'User-Agent',
    error_message VARCHAR(500) COMMENT '错误信息',

    -- 审计字段
    created_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    INDEX idx_bucket_name (bucket_name),
    INDEX idx_user_id (user_id),
    INDEX idx_upload_status (upload_status),
    UNIQUE INDEX uk_bucket_object_time (bucket_name, object_key, start_time),
    INDEX idx_user_status_time (user_id, upload_status, start_time),
    INDEX idx_bucket_status_time (bucket_name, upload_status, start_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='上传记录表';
```

#### 使用示例
```sql
-- 查询简单上传记录
SELECT * FROM upload_record
WHERE upload_type = 'SIMPLE'
  AND upload_status = 'SUCCESS';

-- 查询分片上传记录
SELECT * FROM upload_record
WHERE upload_type = 'MULTIPART'
  AND multipart_upload_id IS NOT NULL
  AND upload_status = 'IN_PROGRESS';

-- 统计上传成功率
SELECT
    upload_type,
    COUNT(*) as total,
    SUM(CASE WHEN upload_status = 'SUCCESS' THEN 1 ELSE 0 END) as success_count,
    SUM(CASE WHEN upload_status = 'FAILED' THEN 1 ELSE 0 END) as failed_count,
    ROUND(SUM(CASE WHEN upload_status = 'SUCCESS' THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as success_rate
FROM upload_record
WHERE start_time >= DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY upload_type;
```

### 4. object_metadata（对象元数据表）

#### 修改说明
- ✅ **添加组合索引**: 优化查询性能
- 💡 **可选：添加sha256字段**: 用于文件去重

#### SQL修改语句
```sql
-- ============================================
-- 步骤1: 添加组合索引（必须执行）
-- ============================================
CREATE INDEX idx_bucket_status
ON object_metadata(bucket_name, status);

CREATE INDEX idx_user_status
ON object_metadata(created_by, status);

CREATE INDEX idx_bucket_user
ON object_metadata(bucket_name, created_by, status);

CREATE INDEX idx_upload_time
ON object_metadata(upload_time DESC);

-- ============================================
-- 步骤2: 添加sha256字段（可选，推荐执行）
-- ============================================
ALTER TABLE object_metadata
    ADD COLUMN file_sha256 VARCHAR(64) COMMENT '文件SHA256哈希值（用于去重）'
    AFTER content_md5,
    ADD INDEX idx_sha256 (file_sha256);
```

#### 修改后的表结构
```sql
CREATE TABLE object_metadata (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    bucket_name VARCHAR(255) NOT NULL COMMENT '存储桶名',
    object_key VARCHAR(1024) NOT NULL COMMENT '对象键',
    version_id VARCHAR(255) COMMENT '版本ID',

    -- 文件信息
    file_size BIGINT COMMENT '文件大小（字节）',
    file_name VARCHAR(500) COMMENT '文件名',
    content_type VARCHAR(255) COMMENT 'MIME类型',
    content_md5 VARCHAR(32) COMMENT 'MD5值',
    file_sha256 VARCHAR(64) COMMENT 'SHA256哈希值（用于去重）',
    crc64 BIGINT COMMENT 'CRC64校验码',

    -- 存储属性
```
