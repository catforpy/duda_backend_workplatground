  ```java
  private Long defaultDuration;
  ```

  ```java
  public STSCredentialsDTO generateSTSCredentials(...) {
      // 1. 从数据库查询Bucket配置
      BucketConfig bucketConfig = bucketConfigMapper.selectByBucketName(bucketName);

      // 2. 检查Bucket状态和配置
      if (!"ACTIVE".equals(bucketConfig.getStatus())) {
          throw new RuntimeException("Bucket状态异常");
      }

      // 3. 解密密钥 ⭐ 关键步骤
      String accessKeyId = aesEncryptUtil.decrypt(bucketConfig.getAccessKeyId());
      String accessKeySecret = aesEncryptUtil.decrypt(bucketConfig.getAccessKeySecret());

      // 4. 使用解密后的密钥调用STS API
      DefaultProfile profile = DefaultProfile.getProfile(
          bucketConfig.getRegion(),  // 从数据库读取
          accessKeyId,
          accessKeySecret
      );

      AssumeRoleRequest request = new AssumeRoleRequest();
      request.setRoleArn(bucketConfig.getStsRoleArn());  // 从数据库读取
      request.setRoleSessionName("duda-file-upload-" + bucketName);

      // 5. 支持外部ID
      if (bucketConfig.getStsExternalId() != null) {
          request.setExternalId(bucketConfig.getStsExternalId());
      }

      // ... 后续逻辑
  }
  ```

## 🔄 工作流程变化

### 修改前的流程

```
用户请求STS
  ↓
从application.properties读取密钥（固定）
  ↓
调用阿里云STS API
  ↓
返回临时凭证
```

**问题**：
- ❌ 只支持公司自己的OSS账号
- ❌ 客户无法使用自己的云存储
- ❌ 密钥明文存储

### 修改后的流程

```
用户请求STS
  ↓
查询bucket_config表（获取该用户的Bucket配置）
  ↓
AES解密密钥
  ↓
调用该用户的云厂商STS API
  ↓
返回临时凭证
```

**优势**：
- ✅ 每个用户独立的云存储配置
- ✅ 支持阿里云、腾讯云、七牛云、MinIO
- ✅ 密钥加密存储，安全可靠

```bash
export ENCRYPTION_AES_KEY=$(openssl rand -base64 32)

# 验证
echo $ENCRYPTION_AES_KEY
```

### Step 2: 更新配置文件

```yaml
# application.yml
encryption:
  aes:
    key: ${ENCRYPTION_AES_KEY}
```

### Step 3: 加密现有密钥（如有）

```java
// 迁移脚本：加密数据库中现有的明文密钥
List<BucketConfig> configs = bucketConfigMapper.selectAll();
for (BucketConfig config : configs) {
    if (config.getAccessKeyId() != null && !isEncrypted(config.getAccessKeyId())) {
        config.setAccessKeyId(aesEncryptUtil.encrypt(config.getAccessKeyId()));
        config.setAccessKeySecret(aesEncryptUtil.decrypt(config.getAccessKeySecret()));
        bucketConfigMapper.update(config);
    }
}
```

## ⚠️ 注意事项

### 1. 密钥安全

- ❌ **不要将AES主密钥硬编码**
- ❌ **不要将AES主密钥提交到Git**
- ✅ **使用环境变量或配置中心**
- ✅ **定期轮换密钥（3-6个月）**

### 2. 数据库迁移

如果数据库中已有明文密钥，需要先加密：
