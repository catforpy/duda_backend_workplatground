# duda-file模块架构设计说明

> **更新时间**: 2025-03-13
> **版本**: v1.0
> **状态**: 设计中

## 1. 模块概述

duda-file是一个统一的文件管理服务模块,支持多云存储(阿里云OSS、腾讯云COS、七牛云Kodo、MinIO等),提供Bucket管理、文件上传下载、目录操作等完整功能。

## 2. 模块划分

### 2.1 三层架构

```
┌─────────────────────────────────────────────────────┐
│         duda-file-interface (接口定义层)              │
│  - 对外暴露的Dubbo服务接口                             │
│  - DTO、枚举、异常定义                                 │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│          duda-file-api (业务抽象层)                   │
│  - 多云适配器(StorageService)                         │
│  - 业务管理器(BucketManager、ObjectManager)           │
│  - 策略模式实现                                        │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│        duda-file-provider (Dubbo服务实现层)           │
│  - Dubbo服务实现                                      │
│  - 数据库访问(Mapper)                                 │
│  - 配置管理                                           │
└─────────────────────────────────────────────────────┘
```

### 2.2 模块职责

| 模块 | 职责 | 依赖 |
|------|------|------|
| **duda-file-interface** | 定义对外暴露的Dubbo服务接口、DTO、枚举、异常 | dubbo-api |
| **duda-file-api** | 多云适配器实现、业务逻辑封装 | interface, 云厂商SDK |
| **duda-file-provider** | Dubbo服务实现、数据库访问、配置管理 | api, interface, MyBatis |

## 3. 核心设计模式

### 3.1 适配器模式(Adapter Pattern)

**目的**: 屏蔽不同云厂商的API差异,提供统一的存储服务接口。

**实现**:
```
StorageService (统一接口)
    ├── AliyunOSSAdapter (阿里云OSS实现)
    ├── TencentCOSAdapter (腾讯云COS实现)
    ├── QiniuKodoAdapter (七牛云Kodo实现)
    └── MinIOAdapter (MinIO实现)
```

**优势**:
- 业务代码无需关心底层云厂商差异
- 切换云厂商只需更换适配器
- 易于扩展新的云存储服务

### 3.2 工厂模式(Factory Pattern)

**目的**: 根据存储类型创建对应的适配器实例。

**实现**:
```
StorageAdapterFactory
    ├── createAdapter(StorageType, ApiKeyConfig)
    ├── createAliyunOSSAdapter()
    ├── createTencentCOSAdapter()
    └── ...
```

### 3.3 策略模式(Strategy Pattern)

**目的**: 根据不同场景选择不同的上传/下载策略。

**实现**:
```
UploadStrategy (上传策略)
    ├── SimpleUploadStrategy (简单上传)
    ├── MultipartUploadStrategy (分片上传)
    └── AppendUploadStrategy (追加上传)
```

### 3.4 门面模式(Facade Pattern)

**目的**: 为复杂的子系统提供简化的统一接口。

**实现**:
```
BucketService (门面)
    └── BucketManager (业务管理器)
            └── StorageService (适配器)
```

## 6. 数据流向

### 6.1 创建Bucket流程

```
用户/客户端
    ↓
[BucketServiceImpl.createBucket()] (Dubbo服务实现)
    ↓
[BucketManager.createBucket()] (业务管理器)
    ↓
1. 验证请求参数
2. 生成Bucket名称
3. 检查配额
4. 检查权限
    ↓
[StorageService.createBucket()] (适配器接口)
    ↓
[AliyunOSSAdapter.createBucket()] (具体适配器实现)
    ↓
[阿里云OSS SDK] (云厂商API)
    ↓
保存Bucket配置到数据库
    ↓
返回BucketDTO
```

### 6.2 上传文件流程

```
用户/客户端
    ↓
[UploadServiceImpl.simpleUpload()] (Dubbo服务实现)
    ↓
1. 选择上传策略(简单上传/分片上传)
2. 验证文件类型和大小
3. 检查存储配额
4. 检查权限
    ↓
[StorageService.uploadObject()] (适配器接口)
    ↓
[AliyunOSSAdapter.uploadObject()] (具体适配器实现)
    ↓
[阿里云OSS SDK] (云厂商API)
    ↓
保存上传记录到数据库
    ↓
返回UploadResultDTO(包含ETag、URL等)
```

### 6.3 下载文件流程

```
用户/客户端
    ↓
[DownloadServiceImpl.download()] (Dubbo服务实现)
    ↓
1. 验证权限
2. 检查对象是否存在
3. 选择下载策略(简单下载/断点续传)
    ↓
[StorageService.downloadObject()] (适配器接口)
    ↓
[AliyunOSSAdapter.downloadObject()] (具体适配器实现)
    ↓
[阿里云OSS SDK] (云厂商API)
    ↓
记录下载统计
    ↓
返回DownloadResultDTO(包含输入流或URL)
```

## 7. 多租户设计

### 7.1 数据隔离

- **Bucket级别隔离**: 每个用户可以创建自己的Bucket
- **目录级别隔离**: 在共享Bucket中通过前缀隔离用户数据
- **权限控制**: 基于userId和userType的权限验证

### 7.2 配额管理

```java
// Bucket配额
{
    "maxSize": 1073741824,      // 最大1GB
    "maxCount": 1000,            // 最多1000个文件
    "currentSize": 536870912,    // 已使用512MB
    "currentCount": 500          // 已上传500个文件
}
```

### 7.3 API密钥管理

- **平台密钥**: 配置在Nacos,所有用户共享
- **用户密钥**: 存储在数据库,每个用户独立
- **加密存储**: 使用AES-256-GCM加密

## 8. 安全设计

### 8.1 权限控制

```java
// Bucket权限
public enum BucketPermission {
    OWNER,      // 所有者
    READ,       // 只读
    WRITE,      // 读写
    ADMIN       // 管理员
}

// Object权限
public enum ObjectPermission {
    PRIVATE,        // 私有
    PUBLIC_READ,    // 公共读
    PUBLIC_WRITE    // 公共读写
}
```

### 8.2 数据安全

- **服务端加密**: 支持SSE-OSS和SSE-KMS
- **传输加密**: 强制使用HTTPS(TLS 1.2+)
- **完整性校验**: 支持MD5和CRC64
- **内容检测**: 可选的内容安全检测和病毒检测

### 8.3 API密钥加密

```java
// AES-256-GCM加密
ApiKeyEncryptionUtil.encrypt(accessKeyId, accessKeySecret);

// 存储到数据库
bucket_config.api_key_encrypted = "encrypted_value";
bucket_config.api_key_iv = "iv_value";
```

## 9. 性能优化

### 9.1 分片上传

- **文件大小阈值**: 超过100MB自动使用分片上传
- **分片大小**: 自动计算(建议1MB-5GB)
- **并发上传**: 支持多线程并发上传分片

### 9.2 断点续传

- **上传记录**: 保存未完成的上传任务
- **分片记录**: 记录已上传的分片
- **恢复机制**: 支持从中断点继续上传

### 9.3 CDN加速

- **自定义域名**: 支持绑定自定义域名
- **CDN配置**: 自动配置CDN加速
- **缓存策略**: 可配置的HTTP缓存头

## 11. 扩展性设计

### 11.1 新增云存储厂商

只需3步:
1. 实现`StorageService`接口
2. 在`StorageAdapterFactory`中添加创建方法
3. 在`StorageType`枚举中添加新类型

### 11.2 新增上传策略

1. 实现`UploadStrategy`接口
2. 在`UploadService`中注册策略
3. 根据文件大小自动选择策略
