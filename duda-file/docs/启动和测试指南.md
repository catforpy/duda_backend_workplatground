## 一、启动前检查清单

### ✅ 1. Nacos配置检查

登录Nacos控制台，确认以下配置已创建：

```
Data ID:    duda-file-provider.yml
Group:      DUDA_FILE_GROUP
配置格式:   YAML
```

**需要确认的配置项**：

| 配置项 | 示例值 | 需要修改 |
|--------|--------|----------|
| `spring.datasource.url` | jdbc:mysql://localhost:3306/duda_file | ✅ 改为你的实际地址 |
| `spring.datasource.username` | root | ✅ 改为你的实际用户名 |
| `spring.datasource.password` | root | ✅ 改为你的实际密码 |
| `dubbo.registry.address` | nacos://120.26.170.213:8848 | ✅ 确认地址正确 |
| `duda.file.encryption.key` | "生成的密钥" | ✅ 必须生成 |
| `duda.file.storage.default-type` | ALIYUN_OSS | 确认存储类型 |
| `duda.file.storage.default-region` | cn-hangzhou | 确认区域 |

**必须执行！**

```bash

# 方法2: 使用在线工具
```

```yaml

  file:
    encryption:
      key: "把你生成的密钥粘贴在这里"
```

### 4.2 创建测试客户端

创建一个简单的测试类来测试上传功能：

**文件**: `duda-file-test/src/test/java/com/duda/file/test/FileServiceTest.java`

```java
package com.duda.file.test;

import com.duda.file.dto.bucket.CreateBucketReqDTO;
import com.duda.file.dto.bucket.BucketDTO;
import com.duda.file.dto.upload.SimpleUploadReqDTO;
import com.duda.file.dto.upload.UploadResultDTO;
import com.duda.file.dto.object.ObjectMetadataDTO;
import com.duda.file.service.BucketService;
import com.duda.file.service.UploadService;
import org.apache.dubbo.config.annotation.DubboReference;
import org.springframework.boot.test.context.SpringBootTest;
import org.junit.jupiter.api.Test;
import java.io.ByteArrayInputStream;
import java.nio.charset.StandardCharsets;

@SpringBootTest
public class FileServiceTest {

    @DubboReference
    private BucketService bucketService;

    @DubboReference
    private UploadService uploadService;

    /**
     * 测试创建Bucket
     */
    @Test
    public void testCreateBucket() {
        CreateBucketReqDTO request = CreateBucketReqDTO.builder()
            .bucketName("test-bucket-" + System.currentTimeMillis())
            .displayName("测试Bucket")
            .region("cn-hangzhou")
            .userId(1L)
            .userType("individual")
            .category("test")
            .build();

        try {
            BucketDTO bucket = bucketService.createBucket(request);
            System.out.println("✅ Bucket创建成功: " + bucket.getBucketName());
        } catch (Exception e) {
            System.err.println("❌ Bucket创建失败: " + e.getMessage());
            e.printStackTrace();
        }
    }

    /**
     * 测试上传文件
     */
    @Test
    public void testUploadFile() {
        String content = "Hello, duda-file! 这是一个测试文件。";
        ByteArrayInputStream inputStream = new ByteArrayInputStream(
            content.getBytes(StandardCharsets.UTF_8)
        );

        ObjectMetadataDTO metadata = ObjectMetadataDTO.builder()
            .contentLength((long) content.getBytes().length)
            .contentType("text/plain")
            .build();

        SimpleUploadReqDTO request = SimpleUploadReqDTO.builder()
            .bucketName("test-bucket")
            .objectKey("test/file-" + System.currentTimeMillis() + ".txt")
            .inputStream(inputStream)
            .metadata(metadata)
            .userId(1L)
            .build();

        try {
            UploadResultDTO result = uploadService.simpleUpload(request);
            System.out.println("✅ 文件上传成功!");
            System.out.println("   ETag: " + result.getETag());
            System.out.println("   文件路径: " + result.getObjectKey());
        } catch (Exception e) {
            System.err.println("❌ 文件上传失败: " + e.getMessage());
            e.printStackTrace();
        }
    }

    /**
     * 测试列出Bucket
     */
    @Test
    public void testListBuckets() {
        try {
            var buckets = bucketService.listBuckets(1L);
            System.out.println("✅ 查询到 " + buckets.size() + " 个Bucket:");
            buckets.forEach(bucket -> {
                System.out.println("   - " + bucket.getBucketName());
            });
        } catch (Exception e) {
            System.err.println("❌ 查询Bucket列表失败: " + e.getMessage());
            e.printStackTrace();
        }
    }
}
```

## 五、常见问题排查

### ❌ 问题1: 无法连接到Nacos

**症状**：
```log
Caused by: com.alibaba.nacos.api.exception.NacosException: Connection refused
```

**解决方案**：
1. 检查Nacos服务是否启动
2. 检查bootstrap.yml中的Nacos地址是否正确
3. 检查网络连接

```bash
# 测试Nacos连接
curl http://120.26.170.213:8848/nacos/
```

### ❌ 问题2: 无法读取Nacos配置

**症状**：
```log
Could not resolve placeholder 'duda.file.encryption.key'
```

**解决方案**：
1. 确认Nacos中已创建 `duda-file-provider.yml`
2. 确认Group是 `DUDA_FILE_GROUP`
3. 确认配置格式是 `YAML`
4. 确认bootstrap.yml中的namespace和group与Nacos一致

### ❌ 问题3: 数据库连接失败

**症状**：
```log
java.sql.SQLException: Access denied for user 'xxx'@'xxx'
```

**解决方案**：
1. 检查数据库地址、端口是否正确
2. 检查用户名、密码是否正确
3. 确认数据库 `duda_file` 已创建
4. 确认数据库用户有权限访问

```bash
# 测试数据库连接
mysql -h 你的数据库IP -P 3306 -u 用户名 -p密码 duda_file
```

### ❌ 问题4: Dubbo服务注册失败

**症状**：
```log
Failed to register dubbo service
```

**解决方案**：
1. 检查Nacos服务是否正常
2. 检查bootstrap.yml中的Group配置
3. 确认端口20884未被占用

```bash
# 检查端口占用
lsof -i :20884
```

### ❌ 问题5: API密钥解密失败

**症状**：
```log
DECRYPTION_FAILED: Failed to decrypt API key
```

**解决方案**：
1. 确认已在Nacos配置 `duda.file.encryption.key`
2. 确认密钥长度至少32字符
3. **重要**: 如果数据库中已有API密钥，它们是明文的，需要：
   - 先删除明文密钥
   - 使用加密后的密钥重新插入

**临时方案**（开发测试）：
```java
// 暂时将加密密钥改为空字符串绕过解密
key: ""
```
