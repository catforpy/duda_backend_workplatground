# 文件管理设计说明

## 概述

基于阿里云OSS的文件管理文档,本文档整理了文件列举、删除、拷贝、解冻、标签管理、软链接、重命名、碎片清理等核心管理功能的实现方式,为后续统一设计提供基础。

## 一、文件列举(ListObjects)

### 1.1 方案概述

**适用场景**:
- 查看Bucket中的所有文件
- 按前缀过滤文件
- 分页查询文件列表
- 递归查询子目录

**核心概念**:
- **MaxKeys**: 单次返回的最大Object数量(最大1000)
- **Marker**: 分页标记,从指定位置开始列举
- **Prefix**: 前缀过滤,只返回指定前缀的Object
- **Delimiter**: 分隔符,用于模拟目录结构(通常为"/")
- **NextMarker**: 下一页的起始位置

### 1.2 简单列举

**Java实现**:
```java
import com.aliyun.oss.OSS;
import com.aliyun.oss.OSSClientBuilder;
import com.aliyun.oss.model.ListObjectsRequest;
import com.aliyun.oss.model.ListObjectsResult;
import com.aliyun.oss.model.OSSObjectSummary;

public class ListObjectsDemo {

    public static void listObjects() {
        String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
        String bucketName = "examplebucket";

        OSS ossClient = new OSSClientBuilder().build(endpoint);

        try {
            // 列举Bucket中的所有Object
            ListObjectsRequest listRequest = new ListObjectsRequest(bucketName);

            // 设置最大返回数量(最大1000)
            listRequest.setMaxKeys(100);

            ListObjectsResult result = ossClient.listObjects(listRequest);

            // 获取Object列表
            List<OSSObjectSummary> objectSummaries = result.getObjectSummaries();

            for (OSSObjectSummary objectSummary : objectSummaries) {
                System.out.println("Object Key: " + objectSummary.getKey());
                System.out.println("Object Size: " + objectSummary.getSize());
                System.out.println("Last Modified: " + objectSummary.getLastModified());
                System.out.println("ETag: " + objectSummary.getETag());
                System.out.println("Storage Class: " + objectSummary.getStorageClass());
                System.out.println("---");
            }

        } finally {
            ossClient.shutdown();
        }
    }
}
```

```java
        e.printStackTrace();

    }
```

**大批量删除策略**:
```java
public static void deleteLargeBatchObjects(List<String> allKeys) {
    String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
    String bucketName = "examplebucket";

    OSS ossClient = new OSSClientBuilder().build(endpoint);

    try {
        // 每批最多1000个文件
        int batchSize = 1000;
        int totalBatches = (allKeys.size() + batchSize - 1) / batchSize;

        for (int i = 0; i < totalBatches; i++) {
            int fromIndex = i * batchSize;
            int toIndex = Math.min(fromIndex + batchSize, allKeys.size());

            List<String> batchKeys = allKeys.subList(fromIndex, toIndex);

            // 删除当前批次
            DeleteObjectsRequest deleteRequest = new DeleteObjectsRequest(bucketName);
            deleteRequest.setKeys(batchKeys);

            DeleteObjectsResult deleteResult = ossClient.deleteObjects(deleteRequest);

            System.out.printf("批次 %d/%d 删除完成, 删除文件数: %d%n",
                i + 1, totalBatches, deleteResult.getDeletedObjects().size());
        }

    } finally {
        ossClient.shutdown();
    }
}
```

### 2.3 删除目录

```java
public static void deleteDirectory() {
    String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
    String bucketName = "examplebucket";
    String directoryPrefix = "images/"; // 目录前缀

    OSS ossClient = new OSSClientBuilder().build(endpoint);

    try {
        // 1. 列举目录下所有文件
        List<String> allKeys = new ArrayList<>();

        String nextMarker = null;
        do {
            ListObjectsRequest listRequest = new ListObjectsRequest(bucketName);
            listRequest.setPrefix(directoryPrefix);
            listRequest.setMaxKeys(1000);

            if (nextMarker != null) {
                listRequest.setMarker(nextMarker);
            }

            ListObjectsResult result = ossClient.listObjects(listRequest);
            allKeys.addAll(result.getObjectSummaries().stream()
                .map(OSSObjectSummary::getKey)
                .collect(Collectors.toList()));

            nextMarker = result.getNextMarker();

        } while (result.isTruncated());

        // 2. 批量删除所有文件
        if (!allKeys.isEmpty()) {
            deleteLargeBatchObjects(allKeys);
            System.out.println("目录删除完成, 共删除 " + allKeys.size() + " 个文件");
        }

    } finally {
        ossClient.shutdown();
    }
}
```

```java
public static void setAdvancedLifecycleRules() {

    try {

        rule3.setPrefix(""); // 应用于所有文件

        rule3.setExpirationDays(1);
        rule3.setStatus(RuleStatus.Enabled);
        rules.add(rule3);

        request.setLifecycleRules(rules);
```

## 四、文件解冻(Restore)

### 4.1 方案概述

**适用场景**:
- 归档存储(Archive)、冷归档(Cold Archive)、深度冷归档(Deep Cold Archive)类型的文件无法直接读取
- 需要先解冻才能访问文件内容

**解冻状态**:
1. **冷冻状态**: 初始状态,不可读取
2. **解冻中**: 异步处理中(202 Accepted)
3. **已解冻**: 可读取(200 OK)
4. **副本过期**: 需要重新解冻

### 4.2 归档存储解冻

**特点**: 通常1分钟完成解冻

**副本有效期**: 1~7天

```java
public static void restoreArchiveObject() {
    String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
    String bucketName = "examplebucket";
    String objectName = "archived-object.txt";

    OSS ossClient = new OSSClientBuilder().build(endpoint);

    try {
        // 1. 获取Object元数据,检查存储类型
        ObjectMetadata metadata = ossClient.getObjectMetadata(bucketName, objectName);
        StorageClass storageClass = metadata.getObjectStorageClass();

        if (storageClass == StorageClass.Archive) {
            System.out.println("检测到归档存储类型,开始解冻...");

            // 2. 发起解冻请求
            ossClient.restoreObject(bucketName, objectName);

            // 3. 等待解冻完成
            do {
                System.out.println("等待解冻完成...");
                Thread.sleep(1000); // 每秒检查一次

                metadata = ossClient.getObjectMetadata(bucketName, objectName);
            } while (!metadata.isRestoreCompleted());

            System.out.println("解冻完成!");
        }

        // 4. 获取解冻后的文件
        OSSObject ossObject = ossClient.getObject(bucketName, objectName);
        ossObject.getObjectContent().close();

    } catch (Exception e) {
        e.printStackTrace();
    } finally {
        ossClient.shutdown();
    }
}
```

### 4.3 冷归档存储解冻

**特点**:
- 高优先级(Expedited): 通常1小时内完成
- 标准(Standard): 通常2~5小时完成
- 批量(Bulk): 通常5~12小时完成

**副本有效期**: 1~365天

```java
public static void restoreColdArchiveObject() {
    String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
    String bucketName = "examplebucket";
    String objectName = "cold-archived-object.txt";

    OSS ossClient = new OSSClientBuilder().build(endpoint);

    try {
        // 获取Object元数据
        ObjectMetadata metadata = ossClient.getObjectMetadata(bucketName, objectName);
        StorageClass storageClass = metadata.getObjectStorageClass();

        if (storageClass == StorageClass.ColdArchive) {
            System.out.println("检测到冷归档存储类型,开始解冻...");

            // 设置解冻参数
            RestoreRequestParameters parameters = new RestoreRequestParameters();
            parameters.setTier(Tier.Standard); // 优先级: Expedited, Standard, Bulk
            parameters.setDays(7); // 副本有效期: 7天

            // 发起解冻请求
            ossClient.restoreObject(bucketName, objectName, parameters);

            System.out.println("解冻请求已提交,预计2~5小时完成");

            // 等待解冻完成(可能需要数小时)
            // 实际应用中建议使用异步任务+数据库记录
            do {
                Thread.sleep(60000); // 每分钟检查一次
                metadata = ossClient.getObjectMetadata(bucketName, objectName);

                if (metadata.isRestoreCompleted()) {
                    System.out.println("解冻完成!");
                    break;
                }

                System.out.println("解冻中...");

            } while (true);
        }

    } catch (Exception e) {
        e.printStackTrace();
    } finally {
        ossClient.shutdown();
    }
}
```

### 4.5 解冻配额说明

**冷归档配额**:
- 平均每秒500个Object
- 每天总解冻配额: 100~120 TB

**深度冷归档配额**:
- 平均每秒100个Object
- 每天总解冻配额: 10~15 TB

**超出配额**: 请求排入队列,解冻时间可能延长

### 4.6 异步解冻方案

**场景**: 大量文件需要解冻,避免阻塞

```java
@Service
public class AsyncRestoreService {

    @Autowired
    private OSSClient ossClient;

    @Autowired
    private RestoreTaskRepository restoreTaskRepository;

    /**
     * 异步解冻文件
     */
    @Async
    public void asyncRestore(String bucketName, String objectName) {
        try {
            // 1. 创建解冻任务记录
            RestoreTask task = new RestoreTask();
            task.setBucketName(bucketName);
            task.setObjectName(objectName);
            task.setStatus("PENDING");
            task.setCreatedAt(LocalDateTime.now());

            restoreTaskRepository.save(task);

            // 2. 发起解冻请求
            ossClient.restoreObject(bucketName, objectName);

            // 3. 更新任务状态
            task.setStatus("RESTORING");
            restoreTaskRepository.save(task);

            // 4. 定时任务会检查解冻状态

        } catch (Exception e) {
            task.setStatus("FAILED");
            task.setError(e.getMessage());
            restoreTaskRepository.save(task);
        }
    }

    /**
     * 检查解冻状态(定时任务调用)
     */
    @Scheduled(fixedDelay = 60000) // 每分钟执行一次
    public void checkRestoreStatus() {
        List<RestoreTask> pendingTasks =
            restoreTaskRepository.findByStatus("RESTORING");

        for (RestoreTask task : pendingTasks) {
            try {
                ObjectMetadata metadata =
                    ossClient.getObjectMetadata(task.getBucketName(), task.getObjectName());
```

## 六、软链接管理

### 6.1 软链接概述

**特点**:
- 类似Windows快捷方式
- 存储目标文件路径名,而非文件内容
- 占用存储容量小(约0.009KB)
- 不支持服务端加密

**访问权限原则**: 最小权限原则
- 需要同时具有软链接权限和目标文件权限
- 软链接为私有 + 目标文件为公共读 = 无法访问
- 软链接为公共读 + 目标文件为私有 = 无法访问

## 七、文件重命名

**重要**: OSS不支持直接重命名文件

**解决方案**:
1. 拷贝文件到新名称
2. 删除原文件

```java
public static void renameObject() {
    String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
    String bucketName = "examplebucket";
    String oldObjectName = "olddir/oldname.txt";
    String newObjectName = "newdir/newname.txt";

    OSS ossClient = new OSSClientBuilder().build(endpoint);

    try {
        // 1. 拷贝文件到新名称
        CopyObjectRequest copyRequest = new CopyObjectRequest(
            bucketName,
            oldObjectName,
            bucketName,
            newObjectName
        );

        CopyObjectResult copyResult = ossClient.copyObject(copyRequest);

        System.out.println("文件拷贝成功, ETag: " + copyResult.getETag());

        // 2. 删除原文件
        ossClient.deleteObject(bucketName, oldObjectName);

        System.out.println("原文件删除成功, 重命名完成");

    } finally {
        ossClient.shutdown();
    }
}
```

**改进方案(原子性重命名)**:
```java
public static void renameObjectAtomically() {
    String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
    String bucketName = "examplebucket";
    String oldObjectName = "olddir/oldname.txt";
    String newObjectName = "newdir/newname.txt";
    String tempObjectName = "temp/" + UUID.randomUUID() + ".tmp";

    OSS ossClient = new OSSClientBuilder().build(endpoint);

    try {
        // 1. 拷贝到临时位置
        ossClient.copyObject(
            bucketName,
            oldObjectName,
            bucketName,
            tempObjectName
        );

        // 2. 验证拷贝成功
        OSSObject tempObject = ossClient.getObject(bucketName, tempObjectName);
        tempObject.getObjectContent().close();

        // 3. 删除原文件
        ossClient.deleteObject(bucketName, oldObjectName);

        // 4. 从临时位置拷贝到目标位置
        ossClient.copyObject(
            bucketName,
            tempObjectName,
            bucketName,
            newObjectName
        );

        // 5. 删除临时文件
        ossClient.deleteObject(bucketName, tempObjectName);

        System.out.println("原子性重命名完成");

    } catch (Exception e) {
        System.err.println("重命名失败: " + e.getMessage());

        // 清理临时文件(如果存在)
        try {
            ossClient.deleteObject(bucketName, tempObjectName);
        } catch (Exception ignored) {
        }

    } finally {
        ossClient.shutdown();
    }
}
```

## 八、删除碎片

### 8.1 碎片概述

**什么是碎片**:
- 分片上传过程中产生的Part
- 未调用CompleteMultipartUpload合并的Part
- 未调用AbortMultipartUpload取消的Part

**费用说明**:
- 碎片占用存储空间
- 按Part的存储类型、大小、时长收取存储费用
- Part无最小计量限制

**重要**: 建议及时完成或取消分片上传,避免产生额外费用

### 8.2 取消分片上传

## 十二、常见问题

### 12.1 删除文件后能否恢复?

**答**: 不能

- 文件删除后无法恢复
- 建议开启版本控制
- 开启版本控制后,删除会创建Delete Marker,可恢复

### 12.2 如何永久删除文件?

**答**:

1. **未开启版本控制**: 直接删除即可永久删除
2. **开启版本控制**:
   - 删除所有版本
   - 或删除Delete Marker

### 12.3 删除目录时提示目录不存在?

**原因**: OSS没有真实的目录,目录是前缀为"/"的Object

**解决**:
```java
// 先列举目录下的所有文件
listRequest.setPrefix("directory/");
listRequest.setDelimiter(""); // 递归列举

// 然后批量删除
deleteObjects(keys);
```

### 12.4 批量删除时部分文件删除失败?

**原因**:
- 文件不存在
- 权限不足
- 网络问题

**解决**:
```java
// 使用DeleteObjectsRequest的quiet模式
// 不会因为部分文件不存在而报错
deleteRequest.setQuiet(true);
```

### 12.5 解冻需要多长时间?

**参考时间**:
- 归档: 通常1分钟
- 冷归档:
  - 高优先级: ~1小时
  - 标准: ~2~5小时
  - 批量: ~5~12小时
- 深度冷归档:
  - 高优先级: ~12小时
  - 标准: ~48小时

**注意**: 实际时间可能有所不同

### 12.6 解冻后能否永久保持可读状态?

**答**: 不能

解冻只是创建临时副本:
- 归档: 最长7天
- 冷归档: 最长365天
- 深度冷归档: 最长365天

**永久解决方案**:
```java
// 拷贝为标准存储类型
CopyObjectRequest copyRequest = new CopyObjectRequest(
    bucketName,
    archivedObject,
    bucketName,
    standardObject
);

ObjectMetadata metadata = new ObjectMetadata();
metadata.setHeader(OSSHeaders.OSS_STORAGE_CLASS, StorageClass.Standard);
copyRequest.setNewObjectMetadata(metadata);

ossClient.copyObject(copyRequest);
```

### 12.7 如何判断Object是否为软链接?

```java
// 获取Object元数据
ObjectMetadata metadata = ossClient.getObjectMetadata(bucketName, objectName);

// 检查是否有x-oss-object-type
String objectType = metadata.getObjectType();
if ("Symlink".equals(objectType)) {
    System.out.println("这是一个软链接");
}
```
