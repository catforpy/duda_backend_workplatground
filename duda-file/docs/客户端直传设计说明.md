# 客户端直传设计说明

## 概述

客户端直传是指移动应用（Android/iOS/Flutter）、微信小程序等客户端直接将文件上传到对象存储OSS,无需经过业务服务器中转。这种方案可以显著降低服务器压力,提升上传速度,改善用户体验。

## 一、为什么选择客户端直传

### 1.1 传统上传方式 vs 客户端直传

**传统上传方式**:
```
客户端 → 业务服务器 → OSS
```

**缺点**:
- ❌ 业务服务器需要处理所有文件流量
- ❌ 上传速度受限于服务器带宽
- ❌ 服务器资源消耗大(CPU、内存、带宽)
- ❌ 需要维护临时文件存储

**客户端直传方式**:
```
客户端 → OSS
       ↓
  业务服务器(仅获取上传凭证)
```

**优点**:
- ✅ **数据安全**: 使用STS临时凭证,权限可控
- ✅ **成本低廉**: 无需大量服务器,只有控制流连接应用服务器
- ✅ **高并发**: 轻松应对高并发访问
- ✅ **弹性扩容**: 存储空间随需扩展
- ✅ **性能优化**: 上传速度更快,用户体验更好
- ✅ **无缝集成**: 支持图片处理、音视频转码等功能

### 1.2 适用场景

| 场景 | 传统上传 | 客户端直传 |
|------|---------|----------|
| 用户上传头像 | ❌ 服务器中转 | ✅ 直接上传 |
| 电商商品图片 | ❌ 服务器中转 | ✅ 直接上传 |
| 社交视频分享 | ❌ 服务器压力大 | ✅ 直接上传 |
| IoT设备数据 | ❌ 服务器瓶颈 | ✅ 直接上传 |
| 企业级大文件 | ❌ 上传慢 | ✅ 高速上传 |

## 二、客户端直传方案对比

### 2.1 三种主流方案

| 方案 | 适用场景 | 安全性 | 复杂度 | 推荐度 |
|------|---------|--------|--------|--------|
| **STS临时凭证** | 移动应用(Android/iOS/Flutter) | ⭐⭐⭐⭐⭐ | 中 | ⭐⭐⭐⭐⭐ |
| **表单上传** | 微信小程序、Web应用 | ⭐⭐⭐⭐ | 中 | ⭐⭐⭐⭐ |
| **预签名URL** | 第三方临时授权 | ⭐⭐⭐⭐ | 低 | ⭐⭐⭐ |

### 2.2 方案选择建议

**移动应用(Android/iOS/Flutter)**:
- ✅ 推荐: **STS临时凭证方案**
- 原因: 权限灵活、安全性高、支持多种操作

**微信小程序**:
- ✅ 推荐: **表单上传方案** (PostObject + V4签名)
- 原因: 微信小程序网络限制、官方推荐方案

**Web应用**:
- ✅ 推荐: **表单上传方案** 或 **预签名URL方案**
- 原因: 实现简单、易于集成

**第三方临时授权**:
- ✅ 推荐: **预签名URL方案**
- 原因: 时效可控、单次授权

## 三、方案一: STS临时凭证方案

### 3.1 方案概述

**适用场景**: 移动应用(Android/iOS/Flutter)

**核心思想**:
1. 客户端向业务服务器请求STS临时凭证
2. 业务服务器调用STS服务获取临时凭证
3. 客户端使用临时凭证直接上传文件到OSS

**优势**:
- ✅ 权限最小化原则(可自定义权限策略)
- ✅ 时效性控制(例如30分钟过期)
- ✅ 无需在客户端存储长期密钥
- ✅ 安全性高(临时凭证泄露影响有限)

**流程图**:
```
┌─────────┐                  ┌──────────┐                 ┌─────┐
│ 客户端  │                  │ 业务服务器 │                 │ STS │
└────┬────┘                  └─────┬────┘                 └──┬──┘
     │                              │                         │
     │ 1. 请求STS凭证                │                         │
     ├─────────────────────────────>│                         │
     │                              │ 2. AssumeRole接口       │
     │                              ├────────────────────────>│
     │                              │                         │
     │                              │ 3. 返回临时凭证          │
     │                              │<────────────────────────┤
     │                              │                         │
     │ 4. 返回临时凭证                │                         │
     │<─────────────────────────────┤                         │
     │                              │                         │
     │ 5. 使用临时凭证上传文件          │                         │
     ├─────────────────────────────────────────────────────>│
     │                              │                         │
     │ 6. 返回上传结果                 │                         │
     │<─────────────────────────────────────────────────────┤
```

### 3.2 前置准备

#### 步骤1: 创建RAM用户(业务服务器使用)

**目的**: 为业务服务器提供调用STS服务的长期凭证

```yaml
# RAM用户配置
用户名称: app-server
访问方式: 使用永久AccessKey访问
权限策略: AliyunSTSAssumeRoleAccess (仅调用AssumeRole接口)
```

**获取信息**:
- AccessKey ID
- AccessKey Secret (只在创建时显示,请妥善保管)

#### 步骤2: 创建RAM角色(用于扮演)

**目的**: 定义临时凭证的权限范围

```yaml
# RAM角色配置
角色类型: 阿里云账号
可信实体: 当前云账号
角色名称: oss-upload-role
```

**获取信息**:
- RoleArn (角色ARN)
  - 格式: `acs:ram::<账号ID>:role/oss-upload-role`

#### 步骤3: 创建自定义权限策略

**目的**: 限制临时凭证的权限(最小授权原则)

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "oss:PutObject",
        "oss:PutObjectTagging"
      ],
      "Resource": "acs:oss:*:*:example-bucket/*"
    },
    {
      "Effect": "Allow",
      "Action": [
        "oss:ListObjects"
      ],
      "Resource": "acs:oss:*.*:example-bucket"
    }
  ]
}
```

**权限说明**:
- `oss:PutObject`: 允许上传文件
- `oss:PutObjectTagging`: 允许设置文件标签
- `oss:ListObjects`: 允许列举文件(可选)

**高级权限策略**(按目录隔离):
```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "oss:PutObject",
      "Resource": "acs:oss:*:*:example-bucket/user-${user_id}/*"
    }
  ]
}
```

#### 步骤4: 为RAM角色授权

**操作**: 将自定义权限策略附加到RAM角色

```
oss-upload-role
  └── 自定义策略: oss-upload-policy
```

### 3.3 业务服务器实现

#### 3.3.1 获取STS临时凭证接口

**接口定义**:
```
GET /api/v1/file/get_sts_token

Response:
{
  "code": 200,
  "message": "success",
  "data": {
    "accessKeyId": "STS.xxx",
    "accessKeySecret": "xxx",
    "securityToken": "xxx",
    "expiration": "2026-03-13T12:00:00Z"
  }
}
```

**Java实现示例**:
```java
@RestController
@RequestMapping("/api/v1/file")
public class StsTokenController {

    // RAM角色ARN
    private static final String ROLE_ARN = "acs:ram::123456789:role/oss-upload-role";

    // STS服务地域
    private static final String REGION_ID = "cn-hangzhou";

    @GetMapping("/get_sts_token")
    public Map<String, Object> getStsToken() {
        try {
            // 1. 创建STS客户端
            DefaultProfile profile = DefaultProfile.getProfile(
                REGION_ID,
                System.getenv("ALIBABA_CLOUD_ACCESS_KEY_ID"),
                System.getenv("ALIBABA_CLOUD_ACCESS_KEY_SECRET")
            );

            IAcsClient client = new DefaultAcsClient(profile);

            // 2. 创建AssumeRole请求
            AssumeRoleRequest request = new AssumeRoleRequest();
            request.setRoleArn(ROLE_ARN);
            request.setRoleSessionName("app-session-" + System.currentTimeMillis());
            request.setDurationSeconds(1800L); // 30分钟

            // 3. 调用AssumeRole接口
            AssumeRoleResponse response = client.getAcsResponse(request);

            // 4. 返回临时凭证
            Map<String, Object> result = new HashMap<>();
            result.put("accessKeyId", response.getCredentials().getAccessKeyId());
            result.put("accessKeySecret", response.getCredentials().getAccessKeySecret());
            result.put("securityToken", response.getCredentials().getSecurityToken());
            result.put("expiration", response.getCredentials().getExpiration());

            return Map.of(
                "code", 200,
                "message", "success",
                "data", result
            );

        } catch (Exception e) {
            log.error("获取STS凭证失败", e);
            return Map.of(
                "code", 500,
                "message", "获取STS凭证失败: " + e.getMessage()
            );
        }
    }
}
```

#### 3.3.2 STS凭证缓存策略

**Redis缓存示例**:
```java
@Service
public class StsTokenService {

    @Autowired
    private RedisTemplate<String, Object> redisTemplate;

    private static final String STS_TOKEN_KEY = "sts:token:";
    private static final long TOKEN_EXPIRE_SECONDS = 1500; // 25分钟(比30分钟短5分钟)

    public Map<String, String> getStsToken(Long userId) {
        String cacheKey = STS_TOKEN_KEY + userId;

        // 1. 尝试从缓存获取
        Map<String, String> cachedToken =
            (Map<String, String>) redisTemplate.opsForValue().get(cacheKey);

        if (cachedToken != null) {
            return cachedToken;
        }

        // 2. 调用STS服务获取新凭证
        Map<String, String> newToken = fetchStsTokenFromStsService(userId);

        // 3. 缓存凭证
        redisTemplate.opsForValue().set(
            cacheKey,
            newToken,
            TOKEN_EXPIRE_SECONDS,
            TimeUnit.SECONDS
        );

        return newToken;
    }
}
```

**实现代码**:
```dart
import 'package:dio/dio.dart';
import 'package:aliyun_oss_dio_wrapper/aliyun_oss_dio_wrapper.dart';

class FileUploadService {
  // 从业务服务器获取STS凭证
  Future<StsCredential> getStsToken() async {
    final response = await Dio().get(
      'https://your-server.com/api/v1/file/get_sts_token',
    );

    final data = response.data['data'];

    return StsCredential(
      accessKeyId: data['accessKeyId'],
      accessKeySecret: data['accessKeySecret'],
      securityToken: data['securityToken'],
      expiration: DateTime.parse(data['expiration']),
    );
  }

  // 上传文件
  Future<String> uploadFile(
    String filePath,
    String objectKey, {
    ProgressCallback? onProgress,
  }) async {
    // 1. 获取STS凭证
    final credential = await getStsToken();

    // 2. 创建OSS客户端
    final ossClient = AliyunOssDioWrapper(
      endpoint: 'https://oss-cn-hangzhou.aliyuncs.com',
      bucketName: 'your-bucket',
      credential: credential,
    );

    // 3. 上传文件
    final result = await ossClient.putObject(
      objectKey: objectKey,
      filePath: filePath,
      onProgress: (count, total) {
        final progress = (count / total * 100).toStringAsFixed(2);
        onProgress?.call(double.parse(progress));
      },
    );

    return result.objectKey;
  }
}

// STS凭证数据模型
class StsCredential {
  final String accessKeyId;
  final String accessKeySecret;
  final String securityToken;
  final DateTime expiration;

  StsCredential({
    required this.accessKeyId,
    required this.accessKeySecret,
    required this.securityToken,
    required this.expiration,
  });

  // 检查凭证是否过期
  bool get isExpired {
    return DateTime.now().isAfter(expiration.subtract(Duration(minutes: 5)));
  }
}
```

#### 3.4.2 Android原生实现示例

**Kotlin代码**:
```kotlin
import com.aliyun.oss.OSSClient
import com.aliyun.oss.common.auth.OSSAuthCredentialsProvider
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.launch

class FileUploadManager(private val context: Context) {

    private val ossClient: OSSClient by lazy {
        // 从业务服务器获取STS凭证
        val credentialProvider = getStsCredentialProvider()

        // 创建OSS客户端
        OSSClient(context, "https://oss-cn-hangzhou.aliyuncs.com", credentialProvider)
    }

    // 获取STS凭证提供者
    private fun getStsCredentialProvider(): OSSAuthCredentialsProvider {
        // 调用业务服务器接口获取STS凭证
        val request = Request.Builder()
            .url("https://your-server.com/api/v1/file/get_sts_token")
            .build()

        val client = OkHttpClient()
        val response = client.newCall(request).execute()
        val jsonData = JSONObject(response.body?.string())

        val accessKeyId = jsonData.getJSONObject("data").getString("accessKeyId")
        val accessKeySecret = jsonData.getJSONObject("data").getString("accessKeySecret")
        val securityToken = jsonData.getJSONObject("data").getString("securityToken")

        return OSSAuthCredentialsProvider(accessKeyId, accessKeySecret, securityToken)
    }

    // 上传文件
    fun uploadFile(
        filePath: String,
        objectKey: String,
        onProgress: (Int) -> Unit,
        onComplete: (String) -> Unit,
        onError: (Exception) -> Unit
    ) {
        CoroutineScope(Dispatchers.IO).launch {
            try {
                val file = File(filePath)

                // 创建上传请求
                val putObjectRequest = PutObjectRequest(
                    "your-bucket",
                    objectKey,
                    file
                )

                // 设置进度回调
                putObjectRequest.setProgressCallback { request, currentSize, totalSize ->
                    val progress = (currentSize * 100 / totalSize).toInt()
                    onProgress(progress)
                }

                // 执行上传
                val putObjectResult = ossClient.putObject(putObjectRequest)

                // 上传成功
                withContext(Dispatchers.Main) {
                    onComplete(putObjectResult.etag)
                }

            } catch (e: Exception) {
                withContext(Dispatchers.Main) {
                    onError(e)
                }
            }
        }
    }
}
```

```java
public Map<String, Object> getStsToken(
```

#### 3.5.2 临时凭证时效控制

**推荐时长**:
- 生产环境: 30分钟
- 测试环境: 60分钟
- 高安全场景: 15分钟

**自动刷新机制**:
```dart
class StsCredentialManager {
  StsCredential? _currentCredential;
  DateTime? _lastRefreshTime;

  Future<StsCredential> getCredential() async {
    // 如果凭证未过期,直接返回
    if (_currentCredential != null && !_currentCredential!.isExpired) {
      return _currentCredential!;
    }

    // 提前5分钟刷新凭证
    if (_lastRefreshTime != null &&
        DateTime.now().difference(_lastRefreshTime!).inMinutes < 25) {
      return _currentCredential!;
    }

    // 刷新凭证
    _currentCredential = await fetchNewCredential();
    _lastRefreshTime = DateTime.now();

    return _currentCredential!;
  }
}
```

### 4.2 微信小程序配置

#### 步骤1: 配置服务器域名白名单

**操作位置**: 微信公众平台 → 开发 → 开发管理 → 服务器域名

**配置项**:
```
request合法域名:
  https://your-server.com

uploadFile合法域名:
  https://examplebucket.oss-cn-hangzhou.aliyuncs.com

downloadFile合法域名:
  https://examplebucket.oss-cn-hangzhou.aliyuncs.com
```

**注意事项**:
- 域名必须使用HTTPS
- 域名不能使用IP地址
- 域名需要在阿里云OSS备案(如果是中国大陆地域)

#### 步骤2: 绑定自定义域名(可选)

**推荐做法**:
```
自定义域名: https://cdn.yourdomain.com
      ↓
CNAME记录: examplebucket.oss-cn-hangzhou.aliyuncs.com
```

```java
@RestController
@RequestMapping("/api/v1/file")
public class PostSignatureController {

    private static final String BUCKET_NAME = "examplebucket";
    private static final String REGION_ID = "cn-hangzhou";

    @GetMapping("/get_post_signature")
    public Map<String, Object> generateSignature(
        @RequestParam(required = false) String prefix,
        @RequestParam(required = false) Long userId
    ) throws Exception {

        // 1. 获取STS临时凭证(使用ECS角色或RAM用户)
        StsCredential credential = getStsCredential();

        // 2. 生成时间戳
        ZonedDateTime now = ZonedDateTime.now(ZoneId.of("UTC"));
        ZonedDateTime expiration = now.plusHours(3); // 3小时有效期

        DateTimeFormatter dateFormatter = DateTimeFormatter.ofPattern("yyyyMMdd");
        DateTimeFormatter timeFormatter = DateTimeFormatter.ofPattern("yyyyMMdd'T'HHmmss'Z'");
        DateTimeFormatter expirationFormatter =
            DateTimeFormatter.ofPattern("yyyy-MM-dd'T'HH:mm:ss.SSS'Z'");

        String dateStr = now.format(dateFormatter);
        String timeStr = now.format(timeFormatter);
        String expirationStr = expiration.format(expirationFormatter);

        // 3. 构建Policy
        Map<String, Object> policy = new HashMap<>();
        policy.put("expiration", expirationStr);

        List<Map<String, String>> conditions = new ArrayList<>();

        // Bucket条件
        conditions.add(Map.of("bucket", BUCKET_NAME));

        // 签名版本条件
        conditions.add(Map.of("x-oss-signature-version", "OSS4-HMAC-SHA256"));

        // 凭证条件
        String credentialStr = String.format("%s/%s/%s/oss/aliyun_v4_request",
            credential.getAccessKeyId(), dateStr, REGION_ID);
        conditions.add(Map.of("x-oss-credential", credentialStr));

        // Token条件
        conditions.add(Map.of("x-oss-security-token", credential.getSecurityToken()));

        // 时间条件
        conditions.add(Map.of("x-oss-date", timeStr));

        // 前缀条件(可选,限制上传目录)
        if (prefix != null) {
            conditions.add(Map.of("starts-with", "$key", prefix));
        }

        policy.put("conditions", conditions);

        // 4. 序列化Policy
        ObjectMapper mapper = new ObjectMapper();
        String jsonPolicy = mapper.writeValueAsString(policy);
        String base64Policy = Base64.encodeBase64String(jsonPolicy.getBytes());

        // 5. 计算签名
        String signature = calculateSignature(
            credential.getAccessKeySecret(),
            dateStr,
            REGION_ID,
            base64Policy
        );

        // 6. 返回签名信息
        return Map.of(
            "code", 200,
            "data", Map.of(
                "security_token", credential.getSecurityToken(),
                "signature", signature,
                "x_oss_date", timeStr,
                "x_oss_credential", credentialStr,
                "x_oss_signature_version", "OSS4-HMAC-SHA256",
                "policy", base64Policy
            )
        );
    }

    /**
     * 计算V4签名
     */
    private String calculateSignature(
        String accessKeySecret,
        String dateStr,
        String region,
        String base64Policy
    ) throws Exception {

        // 1. 计算DateKey
        byte[] dateKey = hmacSha256(
            ("aliyun_v4" + accessKeySecret).getBytes(),
            dateStr
        );

        // 2. 计算DateRegionKey
        byte[] dateRegionKey = hmacSha256(dateKey, region);

        // 3. 计算DateRegionServiceKey
        byte[] dateRegionServiceKey = hmacSha256(dateRegionKey, "oss");

        // 4. 计算SigningKey
        byte[] signingKey = hmacSha256(dateRegionServiceKey, "aliyun_v4_request");

        // 5. 计算Signature
        byte[] signature = hmacSha256(signingKey, base64Policy);

        // 6. 转换为十六进制
        return Hex.encodeHexString(signature);
    }

    /**
     * HMAC-SHA256加密
     */
    private byte[] hmacSha256(byte[] key, String data) throws Exception {
        Mac mac = Mac.getInstance("HmacSHA256");
        SecretKeySpec secretKeySpec = new SecretKeySpec(key, "HmacSHA256");
        mac.init(secretKeySpec);
        return mac.doFinal(data.getBytes());
    }

    /**
     * 获取STS临时凭证
     */
    private StsCredential getStsCredential() {
        // 实现略 - 参考方案一的STS获取逻辑
        return stsTokenService.getStsToken();
    }
}
```

```javascript
class UploadUtils {

  static uploadFile(filePath, objectKey, onProgress) {
    return new Promise((resolve, reject) => {
      // 1. 获取上传参数
      wx.request({
        url: 'https://your-server.com/api/v1/file/get_post_signature',
        method: 'GET',
        data: {
          prefix: 'uploads/' // 可选,指定上传前缀
        },

            reject(new Error('获取上传参数失败'));
            return;
          }

          const signatureData = res.data.data;

          // 2. 构建表单数据
          const formData = {
            'key': objectKey,
            'policy': signatureData.policy,
            'x-oss-signature-version': signatureData.x_oss_signature_version,
            'x-oss-credential': signatureData.x_oss_credential,
            'x-oss-date': signatureData.x_oss_date,
            'x-oss-signature': signatureData.signature,
            'x-oss-security-token': signatureData.security_token,
            'success_action_status': '200'
          };

          // 3. 上传文件到OSS
          wx.uploadFile({
            url: 'https://examplebucket.oss-cn-hangzhou.aliyuncs.com',
            filePath: filePath,
            name: 'file',
            formData: formData,
            success: (uploadRes) => {
              if (uploadRes.statusCode === 200) {
                resolve({
                  objectKey: objectKey,
                  url: `https://examplebucket.oss-cn-hangzhou.aliyuncs.com/${objectKey}`
                });
              } else {
                reject(new Error(`上传失败,状态码: ${uploadRes.statusCode}`));
              }
            },
            fail: (err) => {
              reject(new Error(`上传失败: ${err.errMsg}`));
            }
          });
        },
        fail: (err) => {
          reject(new Error(`请求接口失败: ${err.errMsg}`));
        }
      });
    });
  }

  /**
   * 生成唯一对象键
   */
  static generateObjectKey(fileName) {
    const timestamp = Date.now();
```

```javascript
// 将大文件分片上传
class MultipartUpload {
  async uploadFile(filePath, objectKey, onProgress) {
    // 1. 获取文件信息
    const fileInfo = await this.getFileInfo(filePath);
    const chunkSize = 1024 * 1024; // 1MB
    const totalChunks = Math.ceil(fileInfo.size / chunkSize);

    // 2. 初始化分片上传
    const uploadId = await this.initiateUpload(objectKey);

    // 3. 逐片上传
    for (let i = 0; i < totalChunks; i++) {
      const chunkData = await this.readChunk(filePath, i, chunkSize);

      await this.uploadChunk(uploadId, objectKey, i + 1, chunkData);

      // 更新进度
      const progress = ((i + 1) / totalChunks * 100).toFixed(2);
      onProgress(progress);
    }

    // 4. 完成上传
    await this.completeUpload(uploadId, objectKey);
  }
}
```

## 五、上传回调机制

### 5.1 为什么需要上传回调

**问题场景**:
- 移动应用使用STS凭证上传文件到OSS
- 业务服务器**无法感知**用户上传了哪些数据
- 无法进行数据库记录、权限控制、审核等操作

**解决方案**: 上传回调

### 5.2 上传回调流程

```
┌─────────┐          ┌────┐          ┌──────────┐
│ 客户端  │          │ OSS │          │业务服务器 │
└────┬────┘          ─┬──┘          └─────┬────┘
     │                  │                   │
     │ 1. 上传文件+回调参数 │                   │
     ├─────────────────>│                   │
     │                  │                   │
     │                  │ 2. 回调业务服务器    │
     │                  ├──────────────────>│
     │                  │                   │
     │                  │ 3. 返回处理结果      │
     │                  │<──────────────────┤
     │                  │                   │
     │ 4. 返回上传+回调结果 │                   │
     │<─────────────────┤                   │
```

### 5.3 回调参数配置

#### 5.3.1 系统变量

| 变量 | 含义 | 示例值 |
|------|------|--------|
| `${bucket}` | Bucket名称 | `example-bucket` |
| `${object}` | Object名称 | `uploads/image.jpg` |
| `${etag}` | 文件ETag | `"D8E8A1F3..."` |
| `${size}` | 文件大小 | `102400` |
| `${mimeType}` | MIME类型 | `image/jpeg` |
| `${imageInfo.height}` | 图片高度 | `1080` |
| `${imageInfo.width}` | 图片宽度 | `1920` |
| `${imageInfo.format}` | 图片格式 | `JPEG` |

#### 5.3.2 自定义参数

客户端可以自定义参数传递给业务服务器:

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `x:version` | App版本 | `1.0.0` |
| `x:system` | 操作系统版本 | `iOS 15.0` |
| `x:phone` | 手机型号 | `iPhone 12` |
| `x:user_id` | 用户ID | `123456` |
| `x:gps` | GPS信息 | `30.0,120.0` |

### 5.5 业务服务器接收回调

#### 5.5.1 回调接口实现

**Java实现**:
```java
@RestController
@RequestMapping("/api/v1/file")
public class FileCallbackController {

    @PostMapping("/callback")
    public Map<String, Object> handleCallback(
        @RequestBody Map<String, Object> callbackBody,
        @RequestHeader("authorization") String authorization,
        @RequestHeader("x-oss-pub-key-url") String pubKeyUrl
    ) {
        try {
            // 1. 验证回调签名(防止伪造回调)
            if (!verifyCallbackSignature(authorization, pubKeyUrl)) {
                log.warn("回调签名验证失败");
                return Map.of("status", "error");
            }

            // 2. 解析回调数据
            String bucket = (String) callbackBody.get("bucket");
            String object = (String) callbackBody.get("object");
            long size = Long.parseLong(callbackBody.get("size").toString());
            String etag = (String) callbackBody.get("etag");
            String mimeType = (String) callbackBody.get("mimeType");
            String userId = (String) callbackBody.get("userId");

            // 3. 业务处理
            // 3.1 保存文件记录到数据库
            FileRecord fileRecord = new FileRecord();
            fileRecord.setBucketName(bucket);
            fileRecord.setObjectKey(object);
            fileRecord.setFileSize(size);
            fileRecord.setEtag(etag);
            fileRecord.setMimeType(mimeType);
            fileRecord.setUserId(Long.parseLong(userId));
            fileRecord.setCreatedAt(LocalDateTime.now());

            fileRecordService.save(fileRecord);

            // 3.2 异步处理(如图片压缩、视频转码等)
            asyncProcessFile(fileRecord);

            // 4. 返回成功响应
            return Map.of(
                "status", "success",
                "fileId", fileRecord.getId(),
                "url", getFileUrl(object)
            );

        } catch (Exception e) {
            log.error("处理回调失败", e);
            return Map.of("status", "error");
        }
    }

    /**
     * 验证回调签名
     */
    private boolean verifyCallbackSignature(String authorization, String pubKeyUrl) {
        try {
            // 1. 获取OSS公钥
            byte[] publicKeyBytes = getOssPublicKey(pubKeyUrl);
            PublicKey publicKey = KeyFactory.getInstance("RSA")
                .generatePublic(new X509EncodedKeySpec(publicKeyBytes));

            // 2. 验证签名
            Signature signature = Signature.getInstance("MD5withRSA");
            signature.initVerify(publicKey);
            signature.update(authorization.getBytes());

            byte[] signBytes = Base64.getDecoder()
                .decode(authorization.split(":")[1]);

            return signature.verify(signBytes);

        } catch (Exception e) {
            log.error("验证签名失败", e);
            return false;
        }
    }

    /**
     * 获取OSS公钥
     */
    private byte[] getOssPublicKey(String pubKeyUrl) throws Exception {
        // 从OSS获取公钥
        CloseableHttpClient httpClient = HttpClients.createDefault();
        HttpGet httpGet = new HttpGet(pubKeyUrl);

        CloseableHttpResponse response = httpClient.execute(httpGet);
        String publicKeyBase64 = EntityUtils.toString(response.getEntity());

        return Base64.getDecoder().decode(publicKeyBase64);
    }

    /**
     * 异步处理文件
     */
    @Async
    public void asyncProcessFile(FileRecord fileRecord) {
        // 图片压缩
        if (fileRecord.getMimeType().startsWith("image/")) {
            imageService.compressImage(fileRecord);
        }

        // 视频转码
        if (fileRecord.getMimeType().startsWith("video/")) {
            videoService.transcodeVideo(fileRecord);
        }

        // 病毒扫描
        securityService.scanVirus(fileRecord);

        // 内容审核
        moderationService.moderateContent(fileRecord);
    }

    /**
     * 获取文件URL
     */
    private String getFileUrl(String objectKey) {
        return "https://examplebucket.oss-cn-hangzhou.aliyuncs.com/" + objectKey;
    }
}
```

```python
        'url': f'https://examplebucket.oss-cn-hangzhou.aliyuncs.com/{object_key}'
    })

def verify_signature(authorization, pub_key_url):
    """验证回调签名"""
    try:
        # 1. 获取公钥
        response = requests.get(pub_key_url)
        public_key_pem = response.content

        # 2. 解析公钥
        pub_key = rsa.PublicKey.load_pkcs1(public_key_pem)

        # 3. 验证签名
        message = authorization.encode('utf-8')
        signature = base64.b64decode(authorization.split(':')[1])

        return rsa.verify(message, signature, pub_key)

    except Exception as e:
        print(f"验证签名失败: {e}")
        return False

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=9000)
```

## 六、方案三: 预签名URL方案

### 6.1 方案概述

**适用场景**:
- 第三方临时授权上传
- 单次文件分享
- 不方便集成OSS SDK的场景

**特点**:
- ✅ 无需SDK,直接使用HTTP PUT请求
- ✅ 时效性强(URL过期即失效)
- ✅ 安全性好(签名验证)
- ✅ 实现简单

**流程**:
```
客户端请求上传 → 服务端生成预签名URL →
客户端使用URL上传文件到OSS → 返回结果
```

### 6.2 服务端实现

```java
@GetMapping("/get_presigned_url")
public Map<String, Object> getPresignedUrl(
    @RequestParam String objectKey,
    @RequestParam(defaultValue = "3600") Integer expireSeconds
) {
    try {
        // 1. 获取OSS客户端
        OSS ossClient = getOssClient();

        // 2. 设置过期时间
        Date expiration = new Date(System.currentTimeMillis() + expireSeconds * 1000L);

        // 3. 生成预签名URL
        GeneratePresignedUrlRequest request =
            new GeneratePresignedUrlRequest("example-bucket", objectKey, HttpMethod.PUT);
        request.setExpiration(expiration);

        // 可选: 设置请求头
        Map<String, String> headers = new HashMap<>();
        headers.put("Content-Type", "image/jpeg");
        request.setHeaders(headers);

        URL signedUrl = ossClient.generatePresignedUrl(request);

        // 4. 返回URL
        return Map.of(
            "code", 200,
            "data", Map.of(
                "url", signedUrl.toString(),
                "expiration", expiration.getTime()
            )
        );

    } catch (Exception e) {
        log.error("生成预签名URL失败", e);
        return Map.of("code", 500, "message", e.getMessage());
    }
}
```

### 6.3 客户端实现

**Flutter示例**:

### 9.1 STS凭证获取失败

**问题**: 调用AssumeRole接口失败

**原因**:

```json
    }

  "Version": "1"
```

### 9.2 微信小程序上传失败

**问题**: 上传时提示不在以下域名列表中

**原因**: 未配置uploadFile合法域名

**解决方案**:
1. 登录微信公众平台
2. 配置uploadFile合法域名
3. 域名必须使用HTTPS
4. 域名需要在OSS备案(中国大陆)

### 9.3 上传回调未触发

**问题**: 文件上传成功,但业务服务器未收到回调

**原因**:
1. callbackUrl不可公网访问
2. callbackUrl未配置HTTPS
3. 回调接口处理超时

**解决方案**:
1. 确保callbackUrl可公网访问
2. 使用HTTPS
3. 回调接口快速返回,异步处理业务逻辑
