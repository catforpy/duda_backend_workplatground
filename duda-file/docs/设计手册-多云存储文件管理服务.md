# DudaNexus 多云存储文件管理服务设计方案

## 📋 项目概述

**模块名称：** `duda-file` 文件管理服务
**核心功能：** 图片、视频、音频等多媒体文件的云存储管理
**技术架构：** Spring Boot 3.2.0 + Dubbo 3.3.2 + 多云存储适配
**设计原则：** 统一抽象、适配器模式、多云兼容、配置化切换

---

## 🎯 核心设计理念

### 问题背景

不同的云存储服务商（阿里云OSS、腾讯云COS、千牛云Kodo等）提供的功能基本相同，但API接口差异较大：
- 方法名不同
- 参数结构不同
- 返回结果格式不同
- 认证方式不同

如果业务代码直接调用某个云厂商的SDK，会导致：
1. **供应商锁定**：难以切换到其他云存储
2. **代码重复**：每个云厂商都要写一套类似的代码
3. **维护成本高**：API变更需要修改大量业务代码
4. **测试困难**：需要为每个云厂商单独测试

### 解决方案：统一抽象 + 适配器模式

```
┌─────────────────────────────────────────────────────┐
│           业务层（无需关心底层存储差异）               │
│  FileUploadService, FileDownloadService, etc.       │
└─────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────┐
│          统一存储服务接口（StorageService）           │
│  - uploadFile()   - downloadFile()                   │
│  - deleteFile()   - getFileInfo()                   │
│  - generatePresignedUrl() - listFiles()             │
└─────────────────────────────────────────────────────┘
                         │
        ┌────────────────┼────────────────┐
        ▼                ▼                ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ AliyunAdapter│  │ TencentAdapter│  │ QiniuAdapter │
│  阿里云OSS    │  │ 腾讯云COS     │  │ 千牛云Kodo   │
└──────────────┘  └──────────────┘  └──────────────┘
```

**核心优势：**
1. ✅ 业务代码统一，不依赖具体云厂商
2. ✅ 配置化切换，无需修改代码
3. ✅ 支持多云存储并存
4. ✅ 新增云存储只需实现适配器
5. ✅ 便于单元测试和集成测试

#### 2.2 核心业务流程

```java
/**
 * 文件上传服务（统一业务层）
 * 适配器模式：对业务层屏蔽底层存储差异
 */
@Service
public class FileUploadService {

    @Autowired
    private StorageServiceFactory storageServiceFactory;

    /**
     * 用户端直传：生成上传凭证
     */
    public UploadCredentialDTO generateUploadCredential(
            GenerateUploadCredentialReqDTO reqDTO, Long userId) {

        // 1. 选择存储服务（支持多云）
        StorageService storageService = storageServiceFactory.getStorageService();

        // 2. 选择Bucket
        String bucketName = selectBucket(reqDTO.getFileType());

        // 3. 生成文件ID和对象Key
        String fileId = idGeneratorRpc.generateUserId();
        String objectKey = generateObjectKey(fileId, reqDTO.getFileName(), userId);

        // 4. 生成预签名URL（统一接口）
        PresignedUrlResult presignedUrl = storageService.generatePresignedUrl(
            bucketName,
            objectKey,
            3600,  // 1小时有效期
            "PUT"  // 上传
        );

        // 5. 构建返回DTO
        return UploadCredentialDTO.builder()
            .fileId(fileId)
            .uploadUrl(presignedUrl.getUrl())
            .expiration(presignedUrl.getExpiration())
            .bucketName(bucketName)
            .objectKey(objectKey)
            .build();
    }
}
```

### 模块3：云存储适配器实现

#### 3.1 阿里云OSS适配器

```java
/**
 * 阿里云OSS存储服务适配器
 *
 * @Component("aliyunOssStorageService")
 * @ConditionalOnProperty(name = "storage.type", havingValue = "aliyun-oss")
 */
public class AliyunOssStorageServiceAdapter implements StorageService {

    @Override
    public UploadResult uploadFile(UploadRequest request) {
        // 1. 将统一DTO转换为阿里云OSS专用对象
        ObjectMetadata metadata = new ObjectMetadata();
        metadata.setContentLength(request.getContentLength());
        metadata.setContentType(request.getContentType());

        // 2. 调用阿里云OSS API
        PutObjectResult result = ossClient.putObject(
            request.getBucketName(),
            request.getObjectKey(),
            request.getInputStream(),
            metadata
        );

        // 3. 将阿里云结果转换为统一DTO
        return UploadResult.builder()
            .objectKey(request.getObjectKey())
            .etag(result.getETag())
            .contentLength(metadata.getContentLength())
            .build();
    }

    @Override
    public PresignedUrlResult generatePresignedUrl(
            String bucketName, String objectKey,
            Integer expireSeconds, String method) {
        // 调用阿里云API生成预签名URL
        Date expiration = new Date(System.currentTimeMillis() + expireSeconds * 1000);
        URL url = ossClient.generatePresignedUrl(bucketName, objectKey, expiration);

        return PresignedUrlResult.builder()
            .url(url.toString())
            .expiration(expiration.getTime())
            .build();
    }
}
```

### 模块4：存储服务工厂

```java
/**
 * 存储服务工厂
 * 根据配置自动选择对应的存储服务实现
 */
@Component
public class StorageServiceFactory {

    @Autowired
    private ApplicationContext applicationContext;

    @Autowired
    private StorageConfig storageConfig;

    private final Map<String, StorageService> serviceCache = new ConcurrentHashMap<>();

    /**
     * 获取默认存储服务
     */
    public StorageService getStorageService() {
        return getStorageService(storageConfig.getType());
    }

    /**
     * 根据存储类型获取存储服务
     */
    public StorageService getStorageService(String storageType) {
        return serviceCache.computeIfAbsent(storageType, type -> {
            switch (type) {
                case "aliyun-oss":
                    return applicationContext.getBean("aliyunOssStorageService",
                        StorageService.class);
                case "tencent-cos":
                    return applicationContext.getBean("tencentCosStorageService",
                        StorageService.class);
                case "qiniu-kodo":
                    return applicationContext.getBean("qiniuKodoStorageService",
                        StorageService.class);
                default:
                    throw new UnsupportedOperationException(
                        "不支持的存储类型: " + type);
            }
        });
    }
}
```

## 🔐 安全策略

### 1. STS临时凭证策略

只授予最小权限：
- 只允许上传操作
- 限制路径到用户专属目录
- 文件大小限制
- 短期有效期

### 2. 文件类型白名单

限制允许上传的文件类型：
- 图片：jpg, jpeg, png, gif, webp
- 视频：mp4, avi, mov, mkv
- 音频：mp3, wav, aac, flac

### 3. 回调签名验证

验证OSS上传回调的签名，防止伪造请求。
