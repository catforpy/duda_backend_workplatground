# 文件下载设计说明

## 概述

基于阿里云OSS的文件下载文档,本文档整理了文件下载的核心概念、下载方式、预签名URL生成及使用、限定条件下载等内容,为后续统一设计提供基础。

## 一、下载方式总览

### 1.1 下载方式对比

| 下载方式 | 文件大小限制 | 适用场景 | 优缺点 |
|---------|-------------|---------|--------|
| **简单下载** | 不限 | 一次HTTP请求完成的下载 | ✅ 简单快速 ❌ 不支持断点续传 |
| **断点续传下载** | 不限(推荐>5GB) | 网络不稳定、大文件下载 | ✅ 支持断点续传 ❌ 需要Checkpoint文件 |
| **预签名URL下载** | 不限 | 第三方临时授权下载 | ✅ 安全灵活 ❌ URL有时效限制 |
| **限定条件下载** | 不限 | 增量下载、条件判断 | ✅ 节省带宽 ❌ 需要记录ETag/时间 |

### 1.2 下载方式选择建议

| 场景 | 推荐方式 | 理由 |
|------|---------|------|
| 小文件下载(<5GB) | 简单下载 | 实现简单,一次请求完成 |
| 大文件下载(>5GB) | 断点续传下载 | 支持断点续传,节省流量 |
| 网络不稳定 | 断点续传下载 | 自动恢复,避免重复下载 |
| 第三方临时授权 | 预签名URL下载 | 时效可控,无需SDK |
| 增量同步 | 限定条件下载 | 只下载更新的文件 |
| 文件分享 | 预签名URL下载 | 安全分享,可设置过期 |

## 二、简单下载(GetObject)

### 2.1 方案概述

**适用场景**:
- 一次HTTP请求即可完成的下载
- 文件大小不限(推荐<5GB)
- 网络环境稳定

**特点**:
- 实现简单,直接调用GetObject接口
- 支持流式下载
- 支持指定下载范围

**前置条件**:
- 已上传文件到OSS
- 拥有oss:GetObject权限
- 归档/冷归档文件需要先解冻

### 2.2 实现方式

#### 2.2.1 Java SDK实现

**下载到本地文件**:
```java
import com.aliyun.oss.OSS;
import com.aliyun.oss.OSSClientBuilder;
import com.aliyun.oss.model.GetObjectRequest;
import java.io.File;

public class SimpleDownloadDemo {

    public static void downloadToFile() {
        // OSS客户端配置
        String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
        String bucketName = "examplebucket";
        String objectName = "testfolder/exampleobject.txt";
        String localPath = "D:\\localpath\\examplefile.txt";

        OSS ossClient = new OSSClientBuilder().build(endpoint);

        try {
            // 下载Object到本地文件
            // 如果本地文件存在会覆盖,不存在则新建
            ossClient.getObject(
                new GetObjectRequest(bucketName, objectName),
                new File(localPath)
            );

            System.out.println("文件下载成功: " + localPath);

        } catch (Exception e) {
            System.err.println("文件下载失败: " + e.getMessage());
        } finally {
            ossClient.shutdown();
        }
    }
}
```

```java
        } finally {
            ossClient.shutdown();
        }
    }
}
```

#### 2.2.2 REST API实现

**HTTP GET请求**:
```bash
# 基本下载
GET https://examplebucket.oss-cn-hangzhou.aliyuncs.com/exampleobject.txt

# 范围下载(下载前1MB)
GET https://examplebucket.oss-cn-hangzhou.aliyuncs.com/exampleobject.txt
Range: bytes=0-1048575
```

**Java原生HTTP实现**:
```java
import java.io.IOException;
import java.io.InputStream;
import java.net.HttpURLConnection;
import java.net.URL;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;
import java.nio.file.StandardCopyOption;

public class HttpDownloadDemo {

    public static void downloadFile() throws IOException {
        String fileUrl = "https://examplebucket.oss-cn-hangzhou.aliyuncs.com/exampleobject.txt";
        String localPath = "D:\\localpath\\examplefile.txt";

        URL url = new URL(fileUrl);
        HttpURLConnection connection = (HttpURLConnection) url.openConnection();

        try {
            // 设置请求方法
            connection.setRequestMethod("GET");

            // 连接
            connection.connect();

            // 检查响应码
            if (connection.getResponseCode() == HttpURLConnection.HTTP_OK) {
                // 获取输入流
                InputStream inputStream = connection.getInputStream();

                // 保存到本地文件
                Path targetPath = Paths.get(localPath);
                Files.copy(inputStream, targetPath,
                    StandardCopyOption.REPLACE_EXISTING);

                System.out.println("文件下载成功: " + localPath);
            }

        } finally {
            connection.disconnect();
        }
    }
}
```

## 三、断点续传下载

### 3.1 方案概述

**适用场景**:
- 下载大文件(>5GB)
- 网络不稳定,容易中断
- 需要从断点继续下载

**核心特性**:
- 将大文件分成多个小分片并发下载
- 使用Checkpoint文件记录下载进度
- 某分片下载失败时,只需重下载该分片
- 下载完成后,所有分片合并成完整文件

**前置条件**:
- 已上传文件到OSS
- 拥有oss:GetObject权限
- 确保程序对Checkpoint文件有写权限

### 3.2 Checkpoint文件结构

**Checkpoint文件格式**:
```json
{
  "downloadId": "BE2D0BC931BE4DE1B23F339AABFA49EE",
  "bucketName": "examplebucket",
  "key": "exampleobject.jpg",
  "partSize": 1048576,
  "downloadedParts": [
    {"partNumber": 1, "etag": "\"xxx\"", "size": 1048576},
    {"partNumber": 2, "etag": "\"yyy\"", "size": 1048576}
  ]
}
```

**Checkpoint文件位置**:
- 默认位置: 下载文件同级目录
- 文件名: `<filename>.dcp`
- 可自定义路径

### 3.3 实现方式

#### 3.3.1 Java SDK实现

```java
import com.aliyun.oss.OSS;
import com.aliyun.oss.OSSClientBuilder;
import com.aliyun.oss.model.*;

public class ResumableDownloadDemo {

    public static void downloadFile() {
        String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
        String bucketName = "examplebucket";
        String objectName = "exampledir/exampleobject.txt";
        String localPath = "D:\\localpath\\examplefile.txt";

        OSS ossClient = new OSSClientBuilder().build(endpoint);

        try {
            // 创建下载请求
            DownloadFileRequest downloadFileRequest =
                new DownloadFileRequest(bucketName, objectName);

            // 设置下载到本地文件的完整路径
            downloadFileRequest.setDownloadFile(localPath);

            // 设置分片大小(1MB)
            downloadFileRequest.setPartSize(1 * 1024 * 1024);

            // 设置并发数(10个任务并发下载)
            downloadFileRequest.setTaskNum(10);

            // 开启断点续传下载
            downloadFileRequest.setEnableCheckpoint(true);

            // 设置Checkpoint文件路径(可选)
            downloadFileRequest.setCheckpointFile(
                "D:\\localpath\\examplefile.txt.dcp"
            );

            // 执行下载
            DownloadFileResult downloadRes = ossClient.downloadFile(downloadFileRequest);

            // 获取文件元数据
            ObjectMetadata metadata = downloadRes.getObjectMetadata();
            System.out.println("ETag: " + metadata.getETag());
            System.out.println("最后修改时间: " + metadata.getLastModified());

            System.out.println("文件下载成功");

        } catch (Exception e) {
            System.err.println("下载失败: " + e.getMessage());
        } finally {
            ossClient.shutdown();
        }
    }
}
```

**参数说明**:

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `partSize` | 分片大小 | 1MB~10MB |
| `taskNum` | 并发数 | 3~10 |
| `enableCheckpoint` | 是否启用断点续传 | true |
| `checkpointFile` | Checkpoint文件路径 | 可选 |

**分片大小选择**:
```java
public long calculateOptimalPartSize(long fileSize) {
    if (fileSize < 100 * 1024 * 1024) { // <100MB
        return 1 * 1024 * 1024; // 1MB
    } else if (fileSize < 1024 * 1024 * 1024) { // <1GB
        return 5 * 1024 * 1024; // 5MB
    } else { // >=1GB
        return 10 * 1024 * 1024; // 10MB
    }
}
```

**并发数选择**:
```java
public int getOptimalConcurrency() {
    // WiFi环境
    if (isWifiConnection()) {
        return 10;
    }
    // 4G/5G环境
    else if (isCellularConnection()) {
        return 5;
    }
    // 其他

        return 3;
    }
}
```

### 3.4 断点续传流程

```
1. 开始下载
   ↓
2. 检查Checkpoint文件是否存在
   ↓
   - 存在: 读取已下载分片信息
   - 不存在: 全新下载
   ↓
3. 计算未下载的分片
   ↓
4. 并发下载未完成的分片
   ↓
5. 更新Checkpoint文件
   ↓
6. 所有分片下载完成?
   ↓
   - 否: 返回步骤4
   - 是: 继续下一步
   ↓
7. 合并所有分片
   ↓
8. 删除Checkpoint文件
   ↓
9. 下载完成
```

### 3.5 异常处理

**Checkpoint文件损坏**:

## 四、预签名URL下载

### 4.1 方案概述

**适用场景**:
- 授权第三方临时下载文件
- 文件分享
- 无需SDK的场景

**特点**:
- 无需SDK,直接使用HTTP GET请求
- 时效性强(URL过期即失效)
- 安全性好(签名验证)
- 支持在线预览

**URL格式**:
```
https://BucketName.Endpoint/Object?签名参数
```

**完整示例**:
```
https://examplebucket.oss-cn-hangzhou.aliyuncs.com/exampleobject.txt?
x-oss-process=image%2Fresize%2Cp_10&
x-oss-date=20241115T095058Z&
x-oss-expires=3600&
x-oss-signature-version=OSS4-HMAC-SHA256&
x-oss-credential=LTAI****************%2F20241115%2Fcn-hangzhou%2Foss%2Faliyun_v4_request&
x-oss-signature=6e7a********************************
```

### 4.2 生成预签名URL

#### 4.2.1 Java SDK实现

**基本实现**:
```java
import com.aliyun.oss.OSS;
import com.aliyun.oss.OSSClientBuilder;
import java.net.URL;
import java.util.Date;

public class PresignedUrlDemo {

    public static String generatePresignedUrl() {
        String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
        String bucketName = "examplebucket";
        String objectName = "exampleobject.txt";

        OSS ossClient = new OSSClientBuilder().build(endpoint);

        try {
            // 设置过期时间(1小时)
            Date expiration = new Date(
                System.currentTimeMillis() + 3600 * 1000L
            );

            // 生成预签名URL(GET方法)
            URL url = ossClient.generatePresignedUrl(
                bucketName,
                objectName,
                expiration
            );

            return url.toString();

        } finally {
            ossClient.shutdown();
        }
    }
}
```

**指定响应头**:
```java
public class PresignedUrlWithHeaders {

    public static String generatePresignedUrlWithHeaders() {
        String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
        String bucketName = "examplebucket";
        String objectName = "exampleobject.txt";

        OSS ossClient = new OSSClientBuilder().build(endpoint);

        try {
            // 设置过期时间
            Date expiration = new Date(
                System.currentTimeMillis() + 3600 * 1000L
            );

            // 创建请求
            GeneratePresignedUrlRequest request =
                new GeneratePresignedUrlRequest(
                    bucketName,
                    objectName,
                    HttpMethod.GET
                );

            request.setExpiration(expiration);

            // 设置响应头
            Map<String, String> headers = new HashMap<>();
            // 设置Content-Type
            headers.put("Content-Type", "text/plain; charset=utf-8");
            // 设置强制下载
            headers.put("Content-Disposition", "attachment");
            request.setHeaders(headers);

            // 生成预签名URL
            URL url = ossClient.generatePresignedUrl(request);

            return url.toString();

        } finally {
            ossClient.shutdown();
        }
    }
}
```

**3. 绑定自定义域名**:

## 五、限定条件下载

### 5.1 方案概述

**适用场景**:
- 应用资源更新(只下载更新的文件)
- 数据同步与增量备份
- 文件共享和同步

**核心思想**:
- 使用条件(最后修改时间、ETag)限制下载
- 条件满足才下载,不满足则返回304或412
- 避免重复下载未更改的文件

### 5.2 限定条件参数

| 参数 | 说明 | 返回条件 |
|------|------|---------|
| **If-Modified-Since** | 如果指定时间早于实际修改时间,返回文件+200 OK; 否则返回304 Not Modified | 增量下载 |
| **If-Unmodified-Since** | 如果指定时间晚于或等于实际修改时间,返回文件+200 OK; 否则返回412 Precondition Failed | 确保文件未修改 |
| **If-Match** | 如果ETag匹配,返回文件+200 OK; 否则返回412 Precondition Failed | 确保文件完整性 |
| **If-None-Match** | 如果ETag不匹配,返回文件+200 OK; 否则返回304 Not Modified | 增量下载 |

### 5.3 实现方式

#### 5.3.1 If-Modified-Since示例

**场景**: 只在文件更新后下载

```java
public class ConditionalDownloadDemo {

    public static void downloadIfModified() {
        String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
        String bucketName = "examplebucket";
        String objectName = "testfolder/exampleobject.txt";
        String localPath = "D:\\localpath\\examplefile.txt";

        // 假设本地文件最后修改时间为2023-09-25 13:27:04
        SimpleDateFormat sdf =
            new SimpleDateFormat("EEE, dd MMM yyyy HH:mm:ss 'GMT'", Locale.US);
        sdf.setTimeZone(TimeZone.getTimeZone("GMT"));

        Date lastModifiedTime = new Date(getLocalFileLastModified(localPath));
        String lastModifiedStr = sdf.format(lastModifiedTime);

        OSS ossClient = new OSSClientBuilder().build(endpoint);

        try {
            // 创建下载请求
            GetObjectRequest request = new GetObjectRequest(
                bucketName,
                objectName
            );

            // 设置If-Modified-Since条件
            request.setModifiedSinceConstraint(lastModifiedTime);

            // 尝试下载
            OSSObject ossObject = ossClient.getObject(request);

            if (ossObject != null) {
                // 文件已更新,下载
                InputStream inputStream = ossObject.getObjectContent();
                Files.copy(
                    inputStream,
                    Paths.get(localPath),
                    StandardCopyOption.REPLACE_EXISTING
                );
                inputStream.close();

                System.out.println("文件已更新,下载成功");
            }

        } catch (OSSException oe) {
            if (oe.getErrorCode().equals("304 Not Modified")) {
                // 文件未修改
                System.out.println("文件未修改,无需下载");
            } else {
                System.err.println("下载失败: " + oe.getErrorMessage());
            }
        } catch (Exception e) {
            System.err.println("下载失败: " + e.getMessage());
        } finally {
            ossClient.shutdown();
        }
    }

    private static long getLocalFileLastModified(String filePath) {
        File file = new File(filePath);
        if (file.exists()) {
            return file.lastModified();
        }
        return 0;
    }
}
```

### 5.5 增量同步场景

**场景**: 定期同步OSS文件到本地

```java
public class IncrementalSyncDemo {

    private Map<String, FileMetadata> localFilesMetadata = new HashMap<>();

    /**
     * 增量同步OSS文件到本地
     */
    public void incrementalSync(
        String bucketName,
        String prefix,
        String localDir
    ) {
        String endpoint = "https://oss-cn-hangzhou.aliyuncs.com";
        OSS ossClient = new OSSClientBuilder().build(endpoint);

        try {
            // 1. 列举OSS文件
            ListObjectsRequest listRequest = new ListObjectsRequest(
                bucketName
            );
            listRequest.setPrefix(prefix);
            listRequest.setMaxKeys(1000);

            ObjectListing objectListing = ossClient.listObjects(listRequest);
            List<OSSObjectSummary> objects = objectListing.getObjectSummaries();

            // 2. 对比本地文件
            for (OSSObjectSummary ossObject : objects) {
                String objectKey = ossObject.getKey();
                String localPath = localDir + "/" +
                    objectKey.substring(prefix.length());

                File localFile = new File(localPath);
                FileMetadata metadata = localFilesMetadata.get(objectKey);

                if (!localFile.exists()) {
                    // 本地不存在,直接下载
                    downloadFile(ossClient, bucketName, objectKey, localPath);
                } else if (metadata == null) {
                    // 无元数据记录,直接下载
                    downloadFile(ossClient, bucketName, objectKey, localPath);
                } else {
                    // 有元数据记录,对比ETag或修改时间
                    if (needUpdate(metadata, ossObject)) {
                        downloadFile(ossClient, bucketName, objectKey, localPath);
                    } else {
                        System.out.println("跳过: " + objectKey);
                    }
                }
            }

        } finally {
            ossClient.shutdown();
        }
    }

    /**
     * 判断是否需要更新
     */
    private boolean needUpdate(
        FileMetadata localMetadata,
        OSSObjectSummary ossObject
    ) {
        // 对比ETag
        return !localMetadata.getEtag().equals(ossObject.getETag());
    }

    /**
     * 下载文件
     */
    private void downloadFile(
        OSS ossClient,
        String bucketName,
        String objectKey,
        String localPath
    ) {
        try {
            ossClient.getObject(
                new GetObjectRequest(bucketName, objectKey),
                new File(localPath)
            );
            System.out.println("已下载: " + objectKey);
        } catch (Exception e) {
            System.err.println("下载失败: " + objectKey);
        }
    }

    /**
     * 文件元数据
     */
    static class FileMetadata {
        private String etag;
        private Date lastModified;

        // getters and setters
    }
}
```

```yaml
duda:
  file:
    download:
      # 简单下载配置
      simple:
        enabled: true
        timeout: 300000  # 5分钟超时

      # 断点续传下载配置
      resumable:
        enabled: true
        default-part-size: 10485760  # 10MB
        default-task-num: 5
        checkpoint-dir: /tmp/checkpoints

      # 预签名URL配置
      presigned-url:
        default-expire: 3600  # 1小时
        max-expire: 604800  # 7天
        custom-domain: https://cdn.yourdomain.com

      # 安全配置
      security:
        referer-allow-list:
          - https://yourdomain.com
        ip-whitelist:
          - 192.168.1.0/24
```

### 10.4 数据库表设计

**download_log** - 下载日志表
```sql
CREATE TABLE `download_log` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `bucket_name` VARCHAR(100) NOT NULL,
  `object_key` VARCHAR(500) NOT NULL,
  `file_size` BIGINT,
  `download_type` VARCHAR(20) NOT NULL,  -- SIMPLE, RESUMABLE, PRESIGNED_URL
  `ip_address` VARCHAR(50),
  `user_agent` VARCHAR(500),
  `status` VARCHAR(20) NOT NULL,
  `error_message` TEXT,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_object_key` (`object_key`),
  KEY `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**presigned_url_record** - 预签名URL记录表
```sql
CREATE TABLE `presigned_url_record` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `bucket_name` VARCHAR(100) NOT NULL,
  `object_key` VARCHAR(500) NOT NULL,
  `url_hash` VARCHAR(64) NOT NULL,
  `expire_time` DATETIME NOT NULL,
  `access_count` INT DEFAULT 0,
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_url_hash` (`url_hash`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_expire_time` (`expire_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```
