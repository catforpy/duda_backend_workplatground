#### 2.2 服务启动调试（修复5个关键问题）

**问题1: 数据源配置未加载**
- 错误: `Failed to configure a DataSource: 'url' attribute is not specified`
- 原因: bootstrap.yml中配置组不匹配（FILE_GROUP vs DUDA_FILE_GROUP）
- 解决: 修改配置组为DUDA_FILE_GROUP

**问题2: Dubbo配置中心连接失败**
- 错误: `Failed to create nacos config service client`
- 原因: Dubbo尝试将registry用作config/metadata center
- 解决: 添加配置禁用
  ```yaml
  use-as-config-center: false
  use-as-metadata-center: false
  ```

**问题3: 端口20880被占用**
- 错误: `Address already in use: 20880` (被Docker占用)
- 解决: 修改为 `port: -1` (自动分配端口)

**问题4: MyBatis类型处理错误**
- 错误: `No typehandler found for property uploadedParts`
- 原因: ResumeUploadRecord.uploadedParts字段类型为List<Integer>，但数据库存储VARCHAR
- 解决: 修改实体字段类型从List<Integer>改为String（JSON格式）

**问题5: Nacos注册中心认证失败**
- 错误: `Failed to create nacos naming service client`
- 原因: 注册中心地址缺少认证参数
- 解决: 添加 `?username=nacos&password=nacos` 到注册地址

**最终结果**: 服务成功启动在端口20881，注册了4个Dubbo服务：
- BucketService
- UploadService
- ObjectService
- DownloadService
